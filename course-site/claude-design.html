§ II  ·  От автораFrom the author

Дизайн-инструмент, в котором не нужно уметь дизайнить — нужно уметь попросить. A design tool where you do not need to know design — you need to know how to ask.

Claude Design — не Figma и не Canva. Это разговор. Вы описываете, что нужно, прикрепляете бриф в .docx или таблицу в .xlsx, при необходимости показываете на свой сайт или кодбазу — Claude собирает дизайн. Дальше вы уточняете: инлайн-комментарии к элементам, прямая правка текста, слайдеры spacing / color / layout.

Те же навыки, что у арт-директора с дизайнером в штате: вы ничего не двигаете сами, но обязаны видеть достаточно, чтобы сказать «контраст слабый», «правый блок тяжелее левого», «нет, эта типографика чужая бренду».

Тьюториал ведёт через все возможности Claude Design на примерах из четырёх ниш: кофейня (открытие малого бизнеса), дизайн-проект квартиры (мудборд → 3D-тур → презентация заказчику), инди-игра (UI меню → лендинг → Steam-страница → дек для инвестора) и финансовый продукт (продолжение основного курса). В каждом этапе — два проекта по инструкции на конкретных треках и два самостоятельных, где трек выбираете вы. К концу курса в портфолио работы по всем четырём.

Принцип каждого этапа: то, что вы делаете самостоятельно, — всегда вариация только что разобранного. Новых концепций в solo-проекте не появится — только новый бриф.

В начале каждого этапа — «Подготовка»: какая подписка нужна (Pro / Max / Team / Enterprise), что включить в настройках организации, какие материалы подготовить заранее.

Claude Design is not Figma and not Canva. It is a conversation. You describe what is needed, attach a .docx brief or an .xlsx sheet, optionally point at a website or codebase — Claude assembles the design. Then you refine: inline comments on elements, direct text editing, sliders for spacing / colour / layout.

The same skills as an art director with a designer on staff: you do not push anything yourself, but you must see enough to say “contrast is weak,” “the right block is heavier than the left,” “no, this typography is foreign to the brand.”

The tutorial walks through every Claude Design capability through examples drawn from four niches: a café (small-business launch), an apartment interior project (mood board → 3D tour → client pitch), an indie game (menu UI → landing → Steam page → investor deck), and a financial product (a continuation of the main course). Each etape carries two guided projects on specific tracks and two solo projects where you pick the track. By the end your portfolio carries work across all four.

Principle of every etape: what you do on your own is always a variation of what we just walked through. No new concepts appear in solo work — only a new brief.

Each etape opens with a “Setup” block: which subscription you need (Pro / Max / Team / Enterprise), what to enable in organisation settings, which materials to prepare in advance.

Every etape ends with a live artefact — a PDF, a PPTX, a Canva link, an HTML folder or a Claude Code handoff bundle. By the end you have 18+ ready design assets in your chosen niche: a café brand, an apartment design project, an indie-game launch, or a financial product.

Solo, you do what used to need a studio, a marketer and a 3D visualiser: a café brand, a 3D apartment tour for the client, an indie-game menu UI, a pitch deck, a social series, a handoff to Claude Code. On one condition: Claude Design assembles the visual, you steer and approve.

§ Что вы освоитеWhat you will master

семь навыков-стержней Claude Designseven core Claude Design skills
i.

Промпт + загрузка контекстаPrompt + context upload

Текстовый бриф, прикреплённые .docx / .pptx / .xlsx, ссылки на кодбазу или захват с сайта. Первая итерация Claude Design — уже близко к нужной.

A text brief, attached .docx / .pptx / .xlsx, codebase pointers, or website captures. Claude Design’s first attempt is already close.

ii.

Brand system из кодаBrand system from code

Автоматическое извлечение токенов, типографики, компонентов из вашей кодбазы или Figma. Один источник истины — для всех будущих ассетов.

Automatic extraction of tokens, typography and components from your codebase or Figma. One source of truth for every future asset.

iii.

Три инструмента уточненияThe three refinement tools

Инлайн-комментарии на элементах, прямая правка текста, слайдеры spacing / color / layout. Когда какой использовать — и почему текстовый промпт не всегда лучший выбор.

Inline comments on elements, direct text editing, sliders for spacing / colour / layout. When to reach for which — and why a text prompt is not always the right tool.

iv.

Прототип vs ассетPrototype vs. asset

Когда делать кликабельный прототип для user testing, а когда — финальный экспорт. Состояния, переходы, пустые экраны. Сценарии тестирования.

When to build a clickable prototype for user testing and when to ship a final export. States, transitions, empty screens. Test scenarios.

v.

Handoff в Claude CodeHandoff to Claude Code

Дизайн упаковывается в bundle и уходит напрямую в Claude Code на имплементацию. Цикл бриф → дизайн → код → деплой замыкается без ручной интеграции.

A design packs into a bundle and goes straight to Claude Code for implementation. The brief → design → code → deploy loop closes with no manual integration.

vi.

Команда и ревьюTeam & review

Org-scoped sharing с тремя уровнями доступа. Групповой разговор с Claude. Сведение комментариев, выбор финальной версии.

Org-scoped sharing with three access levels. A group conversation with Claude. Consolidating comments, picking the final version.

vii.

Pipeline экспортаExport pipeline

PDF, PPTX, Canva, HTML, folder, internal URL. Какой формат — для какой задачи. Что отдать маркетологу, что — разработчику, что — инвестору.

PDF, PPTX, Canva, HTML, folder, internal URL. Which format — for which job. What goes to the marketer, the developer, the investor.
