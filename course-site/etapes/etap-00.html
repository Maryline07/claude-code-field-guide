§ I · ТемыTopics

Что мы пройдём по шагамWhat we will walk through

Что такое Claude Code и чем отличается от ChatGPTWhat Claude Code is and how it differs from ChatGPT

ChatGPT — это диалог в браузере. Вы ему пишете, он вам отвечает текстом. Файлы он трогать не умеет.

Claude Code — это тот же Claude, но живущий в вашем терминале и имеющий доступ к вашим файлам. Он может прочитать ваш файл, изменить, создать новый, запустить программу, открыть страницу, скачать данные из интернета. Если коротко: «программист-стажёр у вас на компьютере».

Поэтому курс называется именно курсом по Claude Code, а не «по промптам в чате»: мы будем создавать работающие продукты, а не только обсуждать их.

Метафора, которую полезно держать в голове весь курс: вы — тимлид. Claude — стажёр. Стажёр умный, быстрый и никогда не устаёт. Но ему нужны: чёткое ТЗ, контекст и ваше «да / нет» на каждом шаге.

ChatGPT is a dialogue in a browser. You type, it replies in text. It cannot touch files.

Claude Code is the same Claude, but living in your terminal with access to your files. It can read your file, edit it, create a new one, run a program, open a page, download data from the web. In short: “a programmer-intern on your computer.”

That is why the course is named after Claude Code, not “prompting in chat”: we build working products, not just talk about them.

A metaphor worth holding in your head all course: you are the tech lead. Claude is the intern. A smart, fast, tireless one. But it needs a clear brief, context, and your “yes / no” at each step.

Терминал и рабочая папкаTerminal and working directory

Терминал — это окно, в которое вы печатаете команды. Оно всегда находится в какой-то папке — именно с ней работает Claude Code.

Минимум команд, которые нужно запомнить:

# где я нахожусь
$ pwd

# перейти в папку
$ cd Documents

# создать новую папку
$ mkdir claude-course

# посмотреть содержимое текущей папки
$ ls

Запомните одно правило: Claude Code видит и трогает только ту папку, в которой вы его запустили. Это безопасно: он не может случайно зайти в ваш «Рабочий стол» или «Документы», если вы запустили его в ~/claude-course.

The terminal is a window where you type commands. It always sits in some folder — and that is the folder Claude Code works on.

The minimum set of commands to remember:

# where am I
$ pwd

# move to a folder
$ cd Documents

# create a new folder
$ mkdir claude-course

# list contents of current folder
$ ls

Remember one rule: Claude Code sees and touches only the folder you launched it in. Safe by design: it cannot reach your “Desktop” or “Documents” if you started it in ~/claude-course.

В каждом случае Claude покажет, что именно собирается сделать, и спросит подтверждения (если только вы не разрешили действовать без спроса — об этом следующая тема).

Make a folder, enter it, launch Claude Code:

$ mkdir ~/Desktop/claude-course
$ cd ~/Desktop/claude-course
$ claude

From here on it is a normal dialogue. Type in plain language:

# prompt 1
> create a file hello.txt with the text "Hello, world"

# prompt 2
> read hello.txt and show me the contents

# prompt 3
> replace "Hello" with "Hey" in hello.txt

# prompt 4
> delete hello.txt

Each time Claude shows you what exactly it plans to do and asks for confirmation (unless you allowed it to act without asking — see the next topic).

Режимы разрешений: что Claude делает сам, а что — со спросомPermission modes: what Claude does on its own and what it asks about

В Claude Code есть три режима, переключаются через Shift + Tab:

  1. default — режим по умолчанию. Claude спрашивает разрешение почти на всё «небезопасное»: запись в файл, удаление, запуск команды.
  2. auto-accept — «можно всё». Claude действует без вопросов. Удобно, когда задача рутинная и вы уверены.
  3. plan mode — Claude только планирует, ничего не трогает. Вы видите план, утверждаете, и только потом он его выполняет. Незаменимо для крупных задач — подробно разберём в этапе 04.

На этом этапе оставайтесь в default. Привыкните видеть, что именно Claude собирается сделать — это полезный навык, который пригодится во всём курсе.

Claude Code has three modes, toggled by Shift + Tab:

  1. default — Claude asks for permission for almost anything “unsafe”: writing to a file, deleting, running a command.
  2. auto-accept — “go ahead.” Claude acts without asking. Useful when the task is routine and you are confident.
  3. plan mode — Claude only plans, touching nothing. You see the plan, approve it, and only then it runs. Indispensable for big tasks — covered in detail in etape 04.

For this etape, stay in default. Get used to seeing what Claude intends to do — a habit that pays back across the whole course.

Безопасность: чего Claude никогда не сделаетSafety: what Claude will never do

Несколько важных вещей, чтобы спать спокойно:

  • Claude Code не выходит из своей рабочей папки. Если вы запустили его в ~/Desktop/claude-course, он не увидит файлы из ~/Documents.
  • Деструктивные действия (rm -rf, форс-удаление, git reset --hard) всегда требуют подтверждения, даже в auto-accept.
  • В интернет он ходит только тогда, когда вы явно об этом просите (или есть подключённый MCP — этап 03).
  • Учётные данные, ключи, пароли — только если вы ему сами их дадите. Никаких «утечек» из вашего компьютера наружу.

Если когда-нибудь Claude собирается сделать что-то, что вам не нравится — просто отвечайте «нет», или нажимайте Esc. Это всегда работает.

A few things to let you sleep easy:

  • Claude Code does not leave its working directory. Started in ~/Desktop/claude-course, it cannot see ~/Documents.
  • Destructive actions (rm -rf, force-delete, git reset --hard) always need confirmation, even in auto-accept.
  • It reaches the internet only when you ask, explicitly (or via a connected MCP — etape 03).
  • Credentials, keys, passwords — only if you hand them in. No silent leaks out of your machine.

If Claude ever plans something you do not like — just say “no” or press Esc. That always works.

§ II · Практика по ходуPractice along the way

Десятиминутный прогон по терминалуA ten-minute terminal warm-up

Прежде чем браться за проекты, сделайте небольшую разминку прямо сейчас. Это занимает 5–10 минут и ставит на место всё, о чём вы только что прочитали.

Before tackling the projects, take a brief warm-up right now. It takes 5–10 minutes and locks in everything you just read.

Прогон № 1 · Создаём, читаем, переименовываем, удаляемRun № 1 · Create, read, rename, delete

  1. Откройте терминал. Создайте папку claude-course на рабочем столе.
  2. Перейдите в неё (cd) и запустите claude.
  3. В диалоге попросите: «создай файл note.txt с тремя моими целями на курс».
  4. Откройте файл в VS Code или Блокноте — убедитесь, что он там.
  5. Попросите Claude: «добавь в конец файла четвёртую цель».
  6. Попросите: «переименуй note.txt в goals.md».
  7. Попросите: «удали этот файл». Подтвердите.
  1. Open the terminal. Create a folder claude-course on your Desktop.
  2. Enter it (cd) and launch claude.
  3. In the dialogue ask: “create a file note.txt with three of my course goals.”
  4. Open the file in VS Code or Notepad — confirm it is there.
  5. Ask Claude: “append a fourth goal to the file.”
  6. Ask: “rename note.txt to goals.md.”
  7. Ask: “delete this file.” Confirm.

Что мы тренируем: обращаться к файлам по имени, видеть подтверждения, не бояться удалять.

What we are training: addressing files by name, seeing confirmations, not fearing deletion.

Прогон № 2 · Переключение режимовRun № 2 · Switching modes

  1. Запустите claude заново.
  2. Попросите: «создай 5 файлов с именами day-1.md … day-5.md, в каждом по одной строке — день недели».
  3. Подтвердите каждое действие — почувствуйте, насколько утомительно.
  4. Нажмите Shift+Tab, перейдите в auto-accept.
  5. Попросите: «удали все файлы day-*.md». Заметьте, что подтверждения теперь не нужны.
  6. Снова Shift+Tab — перейдите в plan mode.
  7. Попросите: «создай структуру папок для нового сайта о кофе». Посмотрите, как Claude рисует план, ничего не делая.
  8. Откажитесь от плана. Вернитесь в default.
  1. Launch claude again.
  2. Ask: “create 5 files named day-1.md … day-5.md, each containing one line — a day of the week.”
  3. Confirm each action — feel how tedious it is.
  4. Press Shift+Tab, enter auto-accept.
  5. Ask: “delete all day-*.md files.” Notice — no confirmations needed.
  6. Shift+Tab again — enter plan mode.
  7. Ask: “design the folder structure for a new coffee website.” Watch Claude draw the plan, touching nothing.
  8. Reject the plan. Return to default.

Что мы тренируем: выбирать режим под задачу. Это пригодится буквально на каждом следующем этапе.

What we are training: picking the mode that fits the task. Useful on literally every following etape.

Цель: лендинг про вашу коллекцию — рецепты, любимые места города, настольные игры, кофейни, что угодно. Минимум 5 карточек, каждая с иллюстрацией.

Goal: a landing page about your collection — recipes, favourite city spots, board games, coffee shops, whatever. At least 5 cards, each with an illustration.

Подсказки

Hints

  • Структура та же, что в «Книжной полке».
  • Если коллекция «без обложек» (например, рецепты) — пусть Claude использует иконки или иллюстрации в одном стиле.
  • Хорошо: добавить наверху небольшой блок «об этой коллекции» в курсиве, как редакторскую заметку.
  • Ещё лучше: попросить тёмную и светлую темы и кнопку переключения.
  • Same structure as “Bookshelf.”
  • If your collection has no “covers” (recipes, places) — have Claude use icons or illustrations in one style.
  • Nice: add a short editorial intro in italics at the top.
  • Even nicer: request a light/dark theme with a toggle.

Готово, если…Done if…
  • на странице минимум 5 карточек одной коллекции;
  • у каждой — иллюстрация (фото, обложка или иконка из одного семейства), название и 1–2 строки описания;
  • сверху есть редакторское вступление в курсиве: «об этой коллекции»;
  • есть переключатель темы (тёмная / светлая) и он работает;
  • на узком окне (телефон) карточки выстраиваются в одну колонку и не ломаются.
  • at least 5 cards of one collection;
  • each — an illustration (photo, cover, or one-family icon), title, 1–2 lines of description;
  • an editorial intro in italics at the top: “about this collection”;
  • a working light/dark toggle;
  • on a narrow window (phone) cards stack to one column and do not break.

Если Claude не умеет открыть страницу — ничего страшного, просто попросите оценить код.
Ask Claude last “Open index.html in the browser (via a web tool or Playwright, if you can) and give me feedback as a first-time visitor: in 10 seconds, is it clear what this collection is and why? What gets in the way?”. If Claude cannot open the page — no problem, just ask it to review the code.

Новые слова, которые встретилисьNew words that came up

Inside the text these words are underlined with dots — hover or tap to see a short tooltip. Here is the full glossary for reinforcement. Every next etape has one like this.

АртефактArtefact
В этом курсе так называется каждая ваша готовая работа, которой можно показать пальцем: HTML-страница, скрипт, набор файлов, ссылка в интернете. После каждого этапа у вас новый артефакт. In this course — any finished work of yours you can point at: an HTML page, a script, a folder, a public URL. Every etape leaves you with a new one.
ДиалогDialogue
Ваша переписка с Claude Code в терминале. То, что вы печатаете после >, и что Claude отвечает. Один диалог обычно соответствует одной задаче. Your conversation with Claude Code in the terminal — what you type after > and what Claude replies. One dialogue usually equals one task.
ПромптPrompt
Формулировка задачи, которую вы отправляете Claude. Хороший промпт содержит цель, ограничения, формат, иногда пример. Этап 01 целиком посвящён навыку писать промпты. The wording of a task you send Claude. A good prompt has a goal, constraints, a format, sometimes an example. Etape 01 is dedicated to writing them.
ИтерацияIteration
Повторный заход по той же задаче с поправками. «Сделай» → «не так, измени» → «теперь добавь» → «верни как было» — это четыре итерации одной задачи. Going through the same task again with corrections. “Do it” → “not like that, change” → “now add” → “undo” — four iterations of one task.
ЛендингLanding page
Одностраничный сайт, посвящённый одной теме — продукту, событию, человеку, коллекции. От английского landing: страница, на которую вы «приземлились» и сразу понимаете, о чём она. A one-page site about one topic — a product, an event, a person, a collection. From landing: the page you “arrive on” and immediately get what it is about.
ФутерFooter
Нижняя часть страницы с контактами, копирайтом, ссылками. От английского footer — нижний колонтитул. The bottom of a page with contacts, copyright, links.
HTMLHTML
Язык, на котором написаны все веб-страницы. В курсе вы его сами не пишете — это делает Claude. Вам достаточно знать: один файл .html = одна веб-страница, открывается в любом браузере. The language all web pages are written in. You do not write it yourself in this course — Claude does. Enough to know: one .html file = one web page, opens in any browser.
VS CodeVS Code
Бесплатный редактор файлов от Microsoft. Открывает любой текстовый файл, удобно показывает структуру кода, сделанного Claude. Не обязателен — подойдёт даже Блокнот — но дальше будет проще именно с ним. A free file editor by Microsoft. Opens any text file, shows the structure of code Claude wrote. Not required (Notepad will do), but life is simpler with it.
Hover-эффектHover effect
Реакция элемента на наведение мыши: подсветка, плавное увеличение, смена цвета. От английского hover — «парить», «нависать». An element’s reaction to mouse-hover: highlight, gentle scale, colour change. From hover.
ТерминалTerminal
Окно, в которое вы печатаете команды напрямую компьютеру. Не путайте с веб-чатом или редактором файлов: в терминале вы разговариваете с операционной системой и с Claude Code. A window where you type commands to your computer. Not the same as a web chat or a file editor: in the terminal you talk to the OS and to Claude Code.
Рабочая папкаWorking directory
Папка, внутри которой запущен Claude Code. Он видит и трогает только её содержимое — не может «выйти» наружу. Это безопасная «песочница» для диалога. The folder Claude Code was launched inside. It sees and touches only its contents — cannot reach outside. A safe sandbox for the dialogue.
npmnpm
Программа, которая ставит другие программы. Через неё устанавливается сам Claude Code: npm install -g @anthropic-ai/claude-code. Появляется автоматически вместе с Node.js. A program that installs other programs. That is how you get Claude Code itself: npm install -g @anthropic-ai/claude-code. Comes automatically with Node.js.
§ VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

Постарайтесь сначала ответить мысленно или вслух — и только потом раскрыть «правильный ответ». После того как сверились, отметьте: знал или не знал. По итогам я подскажу, что повторить, чтобы дальше идти уверенно.

Try answering mentally or out loud first — only then reveal the “right answer.” After comparing, mark it: knew or did not know. From the totals I will tell you what to revisit before moving on.

  1. Чем Claude Code принципиально отличается от ChatGPT в браузере? What is the core difference between Claude Code and ChatGPT in a browser?

    показать ответshow answer

    Claude Code живёт в вашем терминале и имеет доступ к файлам на компьютере. Он может прочитать ваш файл, изменить, создать новый, запустить команду. ChatGPT этого не может — он только обменивается текстом в окне браузера.

    Claude Code lives in your terminal and has access to files on your machine. It can read your file, edit it, create a new one, run a command. ChatGPT cannot — it just exchanges text in a browser window.

  2. Какая команда показывает, в какой папке вы сейчас находитесь? Which command shows the folder you are currently in?

    показать ответshow answer

    pwd — от английского print working directory, «напечатай рабочую папку».

    pwd — for print working directory.

  3. Какой клавишей переключаются режимы разрешений в Claude Code, и какие это режимы? Which key cycles the permission modes in Claude Code, and what are they?

    показать ответshow answer

    Shift + Tab. Циклически: default (со спросом) → auto-accept (без спроса) → plan mode (только план) → снова default.

    Shift + Tab. Cycles through default (asks first), auto-accept (no asking), plan mode (plan only), and back.

  4. Когда полезен plan mode? When is plan mode useful?

    показать ответshow answer

    Когда задача крупная, и вы не уверены, что Claude всё правильно понял. В plan mode он сначала рисует план, ничего не трогая. Вы утверждаете план — и только тогда он его выполняет. Подробно разберём в этапе 04.

    When the task is large and you are not sure Claude got it right. In plan mode it draws a plan first, touching nothing. You approve the plan — and only then it runs. Covered in detail in etape 04.

  5. Если запустить Claude Code в папке ~/claude-course, сможет ли он прочитать файл из ~/Documents? If you launch Claude Code in ~/claude-course, can it read a file from ~/Documents?

    показать ответshow answer

    Нет. Claude Code видит только текущую рабочую папку и её содержимое — не выходит наружу. Это базовая мера безопасности.

    No. Claude Code sees only its current working directory and its contents — it does not reach outside. A baseline safety measure.

  6. Что в этом этапе называлось словом «артефакт»? What does the word “artefact” mean in this etape?

    показать ответshow answer

    Готовая ваша работа, которой можно «показать пальцем». В этапе 00 это — четыре HTML-страницы: карточка обо мне, книжная полка, карточка фильма, ваша коллекция.

    A finished piece of your own work you can point at. In etape 00 — four HTML pages: a card about you, a bookshelf, a film card, your collection.

  7. Почему мы поправляем Claude итерациями, а не пытаемся написать «идеальный» промпт сразу? Why correct Claude in iterations rather than try to write the “perfect” prompt at once?

    показать ответshow answer

    Первая версия почти никогда не совпадает с тем, что вы хотели, — даже у профессионалов. Claude быстрый, и пять-шесть итераций по тридцать секунд работают лучше, чем десять минут шлифовки одного «идеального» промпта.

    The first version almost never matches what you wanted — even for pros. Claude is fast, and five-six iterations of thirty seconds beat ten minutes polishing a single “perfect” prompt.
