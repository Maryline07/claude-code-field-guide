Свои skills: рабочая инструкция, которую Claude сам подхватитCustom skills: a working instruction Claude picks up itself

До этого мы ставили чужие skills через Smithery. Они живут в ~/.claude/skills/ или в .claude/skills/ внутри проекта. Свой skill — это такая же папка, только сделанная вами.

Минимальная структура skill:

.claude/skills/personal-cfo/
├── SKILL.md           # главный файл с описанием и логикой
├── scripts/           # вспомогательные скрипты (опционально)
│   └── compute_kpi.py
├── templates/         # шаблоны (опционально)
│   └── monthly-report.md
└── references/        # доп. знания (опционально)
    └── tax-rates.md

Главный файл — SKILL.md. У него фронтматтер с двумя обязательными полями:

---
name: personal-cfo
description: Используется, когда пользователь просит финансовый отчёт.
  Например: "сделай отчёт за апрель", "посчитай savings rate",
  "сравни траты с прошлым кварталом". Анализирует
  data/operations.csv и выдаёт markdown-отчёт.
---

# Personal CFO

## Когда использовать
- запрос финансового отчёта за период;
- ...

## Структура отчёта
...

Главное про description: это «триггер» skill. Claude читает description и сам решает, когда применить skill, без явной команды от вас. Чем точнее description (примеры формулировок, ключевые слова, контексты) — тем лучше срабатывает.

Skill skill-creator учит Claude писать description правильно. Без него вы получите skill, который придётся вызывать словами «используй skill personal-cfo». С ним — «сделай отчёт за апрель», и Claude сам подхватит skill.

So far we installed others’ skills via Smithery. They live in ~/.claude/skills/ or .claude/skills/ inside a project. A custom skill is the same kind of folder, made by you.

Minimal skill layout:

.claude/skills/personal-cfo/
├── SKILL.md           # main description + logic
├── scripts/           # helper scripts (optional)
│   └── compute_kpi.py
├── templates/         # templates (optional)
│   └── monthly-report.md
└── references/        # extra knowledge (optional)
    └── tax-rates.md

The main file is SKILL.md. Its front matter has two required fields:

---
name: personal-cfo
description: Use when the user asks for a financial report.
  Examples: "build me an april report", "compute savings rate",
  "compare spend to last quarter". Reads
  data/operations.csv and returns a markdown report.
---

# Personal CFO

## When to use
- a financial report for a period;
- ...

## Report structure
...

Key thing about description: it’s the skill’s trigger. Claude reads it and decides on its own when to apply the skill, no explicit command from you. The sharper the description (sample phrasings, keywords, contexts) — the better it triggers.

The skill-creator skill teaches Claude to write descriptions right. Without it you get a skill you must summon by name (“use skill personal-cfo”). With it — “build me an April report,” and Claude pulls the skill itself.

Slash-команды: шорткаты для частых задачSlash commands: shortcuts for frequent tasks

Slash-команды — это просто файлы в .claude/commands/ (для проекта) или ~/.claude/commands/ (глобально). Один файл — одна команда.

Файл .claude/commands/прогноз.md:

---
description: Прогноз сбережений с тремя сценариями
allowed-tools:
  - Bash
  - Read
  - Write
  - WebFetch
---

Сделай прогноз сбережений за период из аргументов.

Аргументы: $ARGUMENTS

Шаги:
1. Прочитай data/operations.csv;
2. Вычисли текущий капитал и месячный денежный поток;
3. Проиграй три сценария (consrv 5%, real 8%, agg 12%);
4. Запиши результат в reports/forecast-$(date +%Y-%m-%d).md;
5. Открой через playwright и сделай скриншот.

После этого в диалоге работает /прогноз 30 лет — Claude применит инструкцию из файла, подставив 30 лет вместо $ARGUMENTS.

Что важно:

  • Slash-команда — это не skill. Команда выполняется только когда вы её вызвали (/прогноз). Skill вызывается автоматически по triggers в description.
  • Slash-команды для проектных рутин кладут в .claude/commands/ и коммитят в git. Команды для всех ваших проектов — в ~/.claude/commands/.
  • allowed-tools — список инструментов, которыми команда может пользоваться. Полезно для безопасности: команда «отчёт» не должна иметь право удалять файлы.

Slash commands are just files in .claude/commands/ (project) or ~/.claude/commands/ (global). One file — one command.

File .claude/commands/forecast.md:

---
description: Three-scenario savings forecast
allowed-tools:
  - Bash
  - Read
  - Write
  - WebFetch
---

Build a savings forecast for the period from arguments.

Arguments: $ARGUMENTS

Steps:
1. Read data/operations.csv;
2. Compute current capital and monthly cash flow;
3. Play three scenarios (consrv 5%, real 8%, agg 12%);
4. Write the result to reports/forecast-$(date +%Y-%m-%d).md;
5. Open via playwright and screenshot.

Now /forecast 30 years in the dialogue triggers the file, substituting 30 years for $ARGUMENTS.

What matters:

  • A slash command is not a skill. A command runs only when you call it (/forecast). A skill triggers automatically by description.
  • Project-wide commands live in .claude/commands/, committed to git. Commands across all projects — in ~/.claude/commands/.
  • allowed-tools — the list of tools the command may use. Useful for safety: a “report” command should not be allowed to delete files.
Hooks: автоматические действия в ответ на событияHooks: automatic actions on events

Hook — это команда (или скрипт), которую Claude Code запускает автоматически в ответ на определённое событие. Настраивается в ~/.claude/settings.json (глобально) или .claude/settings.json (проектно).

Главные события:

  • PreToolUse — перед использованием инструмента (Bash, Edit, Write, и т. д.). Можно заблокировать опасную команду до её запуска.
  • PostToolUse — после использования. Удобно для логирования.
  • Stop — когда Claude закончил отвечать. Можно показать уведомление или открыть результат.
  • SessionStart — при старте сессии. Можно прогреть контекст полезной информацией.
  • SessionEnd — при выходе. Хорошее место для ведения timesheet.
  • UserPromptSubmit — перед обработкой вашего сообщения. Можно перехватить или дополнить контекст.
  • PreCompact — перед /compact. Можно сохранить состояние.

Простой hook — «лог запуска любой Bash-команды»:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date +%Y-%m-%d_%H:%M) bash\" >> ~/.claude/bash.log"
          }
        ]
      }
    ]
  }
}

Более полезный hook — «защита от утечки секретов»: перед каждым git commit сканировать staged-файлы на ключи, и если есть — блокировать.

Главное правило: hooks выполняются автоматически. Если поставите hook, который занимает 5 секунд — каждый запуск Bash будет тормозить на 5 секунд. Делайте hooks быстрыми.

A hook is a command (or script) Claude Code runs automatically on a specific event. Configured in ~/.claude/settings.json (global) or .claude/settings.json (project).

Main events:

  • PreToolUse — before a tool (Bash, Edit, Write, etc.). You can block a dangerous command before it runs.
  • PostToolUse — after the tool. Handy for logging.
  • Stop — when Claude finished answering. Show a notification, open the result.
  • SessionStart — at session start. Pre-warm context with useful info.
  • SessionEnd — on exit. Good for timesheets.
  • UserPromptSubmit — before your message is processed. Can intercept or augment context.
  • PreCompact — before /compact. Save state.

A simple hook — “log every Bash run”:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date +%Y-%m-%d_%H:%M) bash\" >> ~/.claude/bash.log"
          }
        ]
      }
    ]
  }
}

A more useful hook — “secret leak guard”: before every git commit scan staged files for keys, block if any.

Key rule: hooks run automatically. A 5-second hook turns every Bash into a 5-second wait. Keep hooks fast.

Output styles: меняем «голос» ClaudeOutput styles: changing Claude’s “voice”

Claude Code умеет переключаться между «стилями ответа». Встроенные:

  • default — обычный, лаконичный. Делает задачу, без лишних объяснений.
  • explanatory — с вставками «★ Insight» по ходу — объясняет важные решения и нюансы кода. Для учёбы.
  • learning — ещё медленнее: задаёт вам вопросы, оставляет места «доделать самому», помогает учиться через практику.

Переключаются через /output-style.

Можно сделать свой output style. Файл ~/.claude/output-styles/cfo.md:

---
name: cfo
description: Финансовый консультант. Сухо, по фактам, с цифрами и %.
---

Отвечай как финансовый консультант:
- максимум фактов и цифр, минимум размышлений;
- каждая цифра с единицей и периодом;
- никогда не говори «возможно» — либо есть данные, либо нет;
- в конце короткая «итоговая рекомендация» в одно предложение.

После создания файла — /output-style cfo, и все ваши финансовые сессии идут в этом тоне.

Полезно иметь стили под разные задачи: cfo для финансов, tutor для учёбы (с вопросами), terse для быстрых правок.

Claude Code can switch between “answer styles.” Built-ins:

  • default — normal, terse. Does the task, no extras.
  • explanatory — with “★ Insight” inserts — explains key decisions and code nuances. For learning.
  • learning — slower: asks you questions, leaves “fill it in yourself” spots, helps learn by doing.

Switch with /output-style.

You can make a custom output style. File ~/.claude/output-styles/cfo.md:

---
name: cfo
description: Financial advisor. Dry, fact-driven, numbers + %.
---

Reply as a financial advisor:
- max facts and figures, min musing;
- every number with units and period;
- never say "maybe" — either you have data or you don't;
- end with one-sentence "final recommendation".

After the file lands — /output-style cfo, and all your finance sessions adopt that tone.

Handy to keep styles per task: cfo for finance, tutor for learning (with questions), terse for quick edits.

settings.json: «панель управления» Claude Codesettings.json: Claude Code’s “control panel”

Файл ~/.claude/settings.json — центральная конфигурация. Сюда идут:

  • permissions — что разрешено без спроса (например, Bash(npm install:*) для npm-команд);
  • env — переменные окружения, видимые Claude (полезно для ANTHROPIC_API_KEY и т. п.);
  • hooks — описанные выше;
  • statusLine — что показывать в нижней строке Claude Code (модель, рабочую папку, токены);
  • plugins — установленные плагины (этап 09).

Уровни:

  • ~/.claude/settings.json — глобально, во всех ваших проектах;
  • .claude/settings.json в корне проекта — настройки именно этого проекта (приоритетнее глобальных);
  • .claude/settings.local.json — ваши личные настройки этого проекта (НЕ коммитятся в git, обычно в .gitignore).

Хорошая практика: общие настройки команды — в .claude/settings.json (коммитятся), личные — в .claude/settings.local.json (нет).

The ~/.claude/settings.json file is the central config. It holds:

  • permissions — what’s allowed without asking (e.g. Bash(npm install:*) for npm commands);
  • env — environment variables Claude sees (handy for ANTHROPIC_API_KEY etc.);
  • hooks — covered above;
  • statusLine — what shows on Claude Code’s bottom bar (model, working directory, tokens);
  • plugins — installed plugins (etape 09).

Levels:

  • ~/.claude/settings.json — global, all your projects;
  • .claude/settings.json at project root — this project only (overrides global);
  • .claude/settings.local.json — your personal settings for this project (NOT committed, typically in .gitignore).

Good practice: team-shared settings in .claude/settings.json (committed), personal in .claude/settings.local.json (not).

Status line, keybindings и другие тонкие настройкиStatus line, keybindings and other fine settings

Status line — нижняя строка интерфейса. По умолчанию там показано базовое. Можно сделать свою — например, отображать использованные токены, время сессии, текущую ветку git. Настраивается в statusLine-блоке settings.json.

Keybindings — клавиатурные сокращения. Например, чтобы Ctrl+L делал /clear, или Ctrl+P вызывал ваш /прогноз. Настраиваются в ~/.claude/keybindings.json. Skill keybindings-help поможет составить.

Memory — auto-memory из этапа 02. Можно явно указать в settings.json, какие типы записей использовать, какие игнорировать. Полезно, если хотите более «приватный» режим.

Permissions — ваш «список белого». Чем больше операций разрешено без спроса, тем быстрее работа, но тем выше риск. Хороший компромисс: разрешить чтение всего и запись в текущую папку, но запросить разрешение на удаление, push и деплой.

{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Edit",
      "Write",
      "Bash(npm:*)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(git log:*)"
    ],
    "ask": [
      "Bash(git push:*)",
      "Bash(rm:*)"
    ]
  }
}

Status line — the bottom strip of the UI. Default shows the basics. You can customise: tokens used, session time, current git branch. Configured in the statusLine block of settings.json.

Keybindings — keyboard shortcuts. E.g., Ctrl+L for /clear, or Ctrl+P for your /forecast. Configured in ~/.claude/keybindings.json. The keybindings-help skill helps draft them.

Memory — auto-memory from etape 02. You can explicitly set in settings.json which record types to use or ignore. Useful for a more “private” mode.

Permissions — your “allow list.” More operations allowed without asking — faster work, higher risk. A good compromise: allow reading everything and writing to the current folder, but ask before delete, push, deploy.

{
  "permissions": {
    "allow": [
      "Read",
      "Glob",

Что выбрать: skill, slash-команда, hook или style?Which to pick: skill, slash command, hook or style?

Четыре способа кастомизации легко перепутать. Простое правило выбора:

КогдаЧто использовать
Хочу, чтобы Claude сам понимал, когда делать XSkill
Хочу нажать /X и получить результатSlash-команда
Хочу, чтобы X происходил автоматически перед/после событияHook
Хочу поменять тон ответов ClaudeOutput style
Хочу разрешить или запретить инструмент глобальноsettings.json

Один проект может использовать всё одновременно: skill для финансовой аналитики, slash-команды для частых задач, hooks для безопасности и логирования, свой output style для тона ответов.

Four customisation paths are easy to mix up. A simple rule:

WhenUse
I want Claude to know on its own when to do XSkill
I want to press /X and get a resultSlash command
I want X to happen automatically before/after an eventHook
I want to change the tone of Claude’s repliesOutput style
I want to allow or block a tool globallysettings.json

One project can use all of them: a skill for financial analytics, slash commands for frequent tasks, hooks for safety and logging, a custom output style for tone.

> создай мой первый output style:

Three levels: global (~/.claude/), project (.claude/), personal-project (.claude/settings.local.json, not committed).
Permissions
«Список белого» в settings.json: какие инструменты Claude может использовать без спроса. Хороший компромисс — разрешать чтение и запись, но запрашивать удаление, push и деплой. An “allow list” in settings.json: which tools Claude may use without asking. Good compromise — allow read/write, but ask for delete, push and deploy.
PreToolUse / PostToolUse / StopPreToolUse / PostToolUse / Stop
Три самых частых события для hooks. PreToolUse срабатывает до вызова инструмента (можно заблокировать). PostToolUse — после (логирование). Stop — когда Claude закончил отвечать. The three most common hook events. PreToolUse fires before a tool call (you can block). PostToolUse — after (logging). Stop — when Claude finished answering.
Status lineStatus line
Нижняя строка интерфейса Claude Code. Можно настроить через statusLine в settings.json: показать модель, токены, ветку git, текущую папку. The bottom strip of Claude Code’s UI. Configurable via statusLine in settings.json: show model, tokens, git branch, current folder.
Keybindings
Клавиатурные сокращения. Файл ~/.claude/keybindings.json. Полезно для часто используемых slash-команд — например, Ctrl+Shift+F = /forecast. Keyboard shortcuts. File ~/.claude/keybindings.json. Handy for frequent slash commands — e.g. Ctrl+Shift+F = /forecast.
Прогрессивное раскрытие (progressive disclosure)Progressive disclosure
Принцип хорошего skill: основной SKILL.md короткий, а детали (шаблоны, скрипты, ссылки на справочники) лежат в подпапках — Claude читает их только когда нужно. Не грузим контекст «впрок». A good-skill principle: the main SKILL.md is short, while details (templates, scripts, reference links) live in subfolders — Claude reads them only when needed. Don’t bloat the context “just in case.”
§ VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

  1. Чем skill отличается от slash-команды?How does a skill differ from a slash command?

    показать ответshow answer

    Skill срабатывает автоматически, когда Claude видит подходящий запрос (по description). Slash-команда запускается только явно — вы вводите /X. Skill для «пусть Claude сам поймёт», команда — для «я знаю, что хочу сделать».

    Skill fires automatically when Claude sees a matching request (by description). Slash command runs only explicitly — you type /X. Skill for “let Claude figure it out,” command for “I know what I want.”

  2. Что важнее в своём skill: name или description?What matters more in a custom skill: name or description?

    показать ответshow answer

    Description. Это «триггер» skill — Claude читает его и решает, когда применить skill. Чем точнее (5–7 примеров формулировок, ключевых слов, контекстов) — тем стабильнее работает. Name важен только для идентификации в /skills.

    Description. It’s the trigger — Claude reads it and decides when to apply the skill. The sharper (5–7 sample phrasings, keywords, contexts) — the more reliable. Name matters only for the /skills listing.

  3. Когда уместен hook PreToolUse?When does a PreToolUse hook fit?

    показать ответshow answer

    Когда нужно проверить или заблокировать действие до его выполнения. Примеры: блок коммита, если в diff обнаружен ключ; запрет rm -rf; запись лога перед каждым Bash. Преимущество — hook успевает остановить действие, в отличие от PostToolUse.

    When you want to check or block an action before it runs. Examples: block commit if a key shows up in the diff; deny rm -rf; log before every Bash. The advantage — the hook can stop the action, unlike PostToolUse.

  4. Какие три уровня settings.json существуют?What are the three settings.json levels?

    показать ответshow answer

    ~/.claude/settings.json — глобально, для всех проектов. .claude/settings.json в корне проекта — командные настройки, коммитятся в git. .claude/settings.local.json — ваши личные настройки, в .gitignore. Приоритет — от частного к общему: local перебивает project, project перебивает global.

    ~/.claude/settings.json — global, all projects. .claude/settings.json at project root — team settings, committed. .claude/settings.local.json — your personal settings, in .gitignore. Priority — narrowest wins: local beats project beats global.

  5. Что делает «прогрессивное раскрытие» в skill?What does “progressive disclosure” do in a skill?

    показать ответshow answer

    Главный SKILL.md остаётся коротким, а детали (шаблоны, скрипты, справочники) лежат в подпапках. Claude читает их только когда нужно для текущей задачи. Это экономит контекст — skill можно делать большим, но «дешёвым» в применении.

    The main SKILL.md stays short while details (templates, scripts, references) live in subfolders. Claude reads them only when needed for the current task. Saves context — the skill can be large yet cheap to apply.

  6. Зачем у slash-команды поле allowed-tools?Why does a slash command have allowed-tools?

    показать ответshow answer

    Безопасность. Команда «отчёт» не должна иметь права удалять файлы или пушить в git. allowed-tools — явный список инструментов, которыми команда может пользоваться. Если команда попытается выйти за рамки — Claude её остановит.

    Safety. A “report” command should not be allowed to delete files or push to git. allowed-tools — an explicit list the command may use. If it tries to step out — Claude stops it.

  7. У вас тёплая палитра, тёмная тема, лаконичный язык. Что выбрать — output style или CLAUDE.md?You have a warm palette, dark theme, terse voice. Which to use — an output style or CLAUDE.md?

    показать ответshow answer

    Зависит. CLAUDE.md — для настроек проекта (стек, данные, стиль кода). Output style — для тона вашего общения с Claude. Если «тёплая палитра» относится к конкретному сайту — CLAUDE.md. Если «лаконичный язык» относится к вашей привычке общаться с Claude в любых проектах — output style. Идеально — и то, и другое.

    It depends. CLAUDE.md — for project settings (stack, data, code style). Output style — for the tone of your dialogue with Claude. “Warm palette” is project-specific → CLAUDE.md. “Terse voice” is your habit across projects → output style. Best — both.
