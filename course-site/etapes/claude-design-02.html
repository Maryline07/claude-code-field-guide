Не пересказывайте бриф.
Загрузите его. Do not retell the brief.
Upload it.

В Claude Code вы прикрепляли файл через @file. В Claude Design — через панель загрузки в диалоге. Принцип тот же, эффект другой: Claude не читает файл как текст, а структурно парсит и собирает дизайн из неё. На этом этапе превращаем реальные клиентские брифы, таблицы и шаблоны деков в готовый визуал без переписывания.

In Claude Code you attached files via @file. In Claude Design — via the upload panel in the dialogue. Same principle, different effect: Claude does not read the file as text but structurally parses it, then assembles the design from that structure. This etape turns real client briefs, sheets, and deck templates into finished visuals without rewriting.

§ I · ТемыTopics

Что мы пройдём по шагамWhat we will walk through

.docx как бриф: Claude парсит, а не пересказывает.docx as a brief: Claude parses, does not retell

В реальной работе бриф редко приходит в виде текстового промпта. Чаще — .docx или Google Doc от клиента, маркетолога или арт-директора. Десять страниц с заголовками, списками, таблицами тарифов, скриншотами референсов.

Раньше у вас был один путь: переписать его в промпт, теряя структуру. В Claude Design — прикрепляете .docx прямо в диалог, и в промпте говорите коротко:

> собери лендинг по бриф-документу выше.
> формат 1280×800 десктоп + мобильная.
> стиль возьми из 4-го раздела «mood»: scandinavian café aesthetic.
> в hero — тезис из «one-line tagline». в блоке «о нас» — текст из раздела 2.

Claude видит структуру: где заголовок «mood», где раздел «about», где таблица позиций. Можете сослаться на конкретный раздел брифа в промпте — «в hero возьми тезис из раздела 1.2» — Claude находит его сам.

Что хорошо парсится:

  • Заголовки (h1, h2, h3 в Word) → семантическая иерархия
  • Маркированные/нумерованные списки → блоки на странице
  • Таблицы → повторяющиеся структуры (см. следующую тему про .xlsx)
  • Курсив, жирный, ссылки → сохраняются в типографике
  • Изображения внутри документа → используются как референсы

Что теряется:

  • Сложная вёрстка с колонками — Claude видит только текст
  • Кастомные шрифты — заменятся на ближайшие из его типографического словаря
  • Footnotes, endnotes — обычно игнорируются (это редко важно для брифа)
Главное правило: хорошо структурированный документ работает лучше длинного промпта. Если бриф пришёл сплошным текстом без заголовков — уйдите 5 минут на то, чтобы расставить h1/h2 и списки. Окупится 30 минут на первой же итерации.

In real work a brief rarely arrives as a text prompt. More often — a .docx or a Google Doc from a client, marketer, or art director. Ten pages of headings, lists, pricing tables, reference screenshots.

You used to have one path: rewrite it into a prompt, losing structure. In Claude Design — attach the .docx in the dialogue and in the prompt say it short:

> assemble the landing per the brief above.
> format 1280×800 desktop + mobile.
> pull style from section 4 "mood": scandinavian café aesthetic.
> hero — the line from "one-line tagline". in "about us" — text from section 2.

Claude sees the structure: where the “mood” heading sits, where “about” lives, where the items table is. You can refer to a specific section of the brief in the prompt — “hero text from section 1.2” — Claude finds it.

Parses well:

  • Headings (h1, h2, h3 in Word) → semantic hierarchy
  • Bulleted/numbered lists → page blocks
  • Tables → repeating structures (see the .xlsx topic next)
  • Italic, bold, links → preserved in typography
  • Embedded images → used as references

Lost:

  • Complex column layouts — Claude sees only the text
  • Custom fonts — replaced with the nearest from its type library
  • Footnotes / endnotes — usually ignored (rarely matters for a brief)
The headline rule: a well-structured document beats a long prompt. If the brief came as a wall of text without headings — spend five minutes adding h1/h2 and lists. Pays back 30 minutes on the very first iteration.
.xlsx как структурированный контент: таблица → повторяющиеся блоки.xlsx as structured content: table → repeating blocks

Это самая «волшебная» возможность загрузок. Любая таблица — меню, прайс-лист, тикеры портфеля, уровни игры, план комнат, расписание, контактный список — превращается в повторяющиеся визуальные элементы.

Принцип: колонки = поля, строки = повторы.

Пример. Таблица меню кофейни:

| Раздел   | Позиция        | Описание                | Цена  |
|----------|----------------|-------------------------|-------|
| espresso | Espresso       | Один шот, 30 мл         | $3.00 |
| espresso | Doppio         | Двойной, 60 мл          | $3.50 |
| espresso | Macchiato      | С пенкой молока         | $4.00 |
| filter   | V60            | Hand-pour, Эфиопия      | $4.50 |
| filter   | Aeropress      | Под давлением, Колумбия | $4.50 |
| filter   | Cold brew      | 18 часов на холоде      | $5.00 |
| sweets   | Цельнозерновой | Овёс, мёд, изюм         | $3.50 |
| sweets   | Морковный      | Грецкий орех, корица    | $4.50 |
| sweets   | Шоколадный     | 70% Сент-Винсент        | $5.00 |

Загружаете .xlsx с этой таблицей. В промпте:

> меню кофейни «Северный край» по таблице выше.
> формат A4 вертикальный, для печати.
> стиль: Müller-Brockmann.
> группировка: колонка «раздел» — это секции (espresso / filter / sweets).
> в каждой секции — три строки. покажи название, описание, цену.
> цены выровняй по правому краю.

Claude собирает три секции по три позиции, шрифты подбирает по референсу, цены выравнивает по правому краю. Если в следующий раз таблица обновится (добавите cold brew с тропической Эфиопией) — перезагружаете .xlsx, и Claude перерисовывает меню с новой строкой.

Это работает для любого «спискового» дизайна:

  • Кофейня: меню, прайс-лист, акции
  • Финансы: портфель (тикер, цена, % изменение, доля), сравнение фондов
  • Игра: уровни (имя, сложность, описание), список персонажей, инвентарь
  • Квартира: план комнат (комната, площадь, материал, бюджет), смета
  • Универсально: команда (имя, роль, фото), кейсы, контакты, расписание событий

This is the most “magical” upload feature. Any table — menu, price list, portfolio tickers, game levels, floor plan, schedule, contact list — turns into repeating visual elements.

Principle: columns = fields, rows = repeats.

Example. Café menu table:

| Section  | Item            | Description              | Price |
|----------|-----------------|--------------------------|-------|
| espresso | Espresso        | Single shot, 30 ml       | $3.00 |
| espresso | Doppio          | Double, 60 ml            | $3.50 |
| espresso | Macchiato       | With milk foam           | $4.00 |
| filter   | V60             | Hand-pour, Ethiopia      | $4.50 |
| filter   | Aeropress       | Under pressure, Colombia | $4.50 |
| filter   | Cold brew       | 18 hours cold            | $5.00 |
| sweets   | Wholegrain      | Oats, honey, raisins     | $3.50 |
| sweets   | Carrot          | Walnut, cinnamon         | $4.50 |
| sweets   | Chocolate       | 70% Saint-Vincent        | $5.00 |

Upload the .xlsx with this table. In the prompt:

> café menu "Northern Coast" from the table above.
> format A4 vertical, for print.
> style: Müller-Brockmann.
> grouping: the "section" column = section blocks (espresso / filter / sweets).
> each section — three rows. show name, description, price.
> right-align prices.

Claude assembles three three-item sections, picks typography from the reference, right-aligns prices. Next time the table updates (you add a cold brew from a tropical Ethiopia) — re-upload the .xlsx, Claude redraws the menu with the new row.

It works for any “list-based” design:

  • Café: menu, price list, promos
  • Finance: portfolio (ticker, price, % change, weight), fund comparisons
  • Game: levels (name, difficulty, brief), character roster, inventory
  • Apartment: room plan (room, area, material, budget), cost estimate
  • Universal: team (name, role, photo), cases, contacts, event schedule
.pptx как шаблон: стилевая основа.pptx as a template: stylistic foundation

Если у компании есть фирменный шаблон презентаций (логотип в углу, корпоративная палитра, типографика, slide-master) — загружаете его как .pptx, и Claude использует его как стилевую основу для нового контента.

> загруженная презентация — наш фирменный шаблон.
> собери дек на 12 слайдов про новую инвест-стратегию,
> используя те же палитру, шрифты, расположение логотипа,
> структуру (титул / содержание / 8 контентных / итог / контакты).
> контент возьми из приложенного брифа.docx.

Это наследование стиля, а не «редактирование оригинала». Claude не трогает ваш шаблон — он читает его и собирает новые слайды в этой стилистике.

Полезно, когда:

  • Вы работаете в корпорации с жёстким брендгайдом
  • Клиент прислал шаблон и просит «как у нас»
  • У вас есть прошлогодняя удачная презентация, и нужно сделать новую в той же стилистике

Дек как артефакт мы разберём подробно в этапе 06 (питч-дек из .xlsx), здесь — только сам приём загрузки шаблона.

If your company has a corporate slide template (logo in the corner, brand palette, type, slide-master) — upload it as .pptx, and Claude uses it as the stylistic foundation for new content.

> the uploaded deck is our corporate template.
> produce a 12-slide deck about a new investment strategy
> using the same palette, fonts, logo placement,
> structure (title / agenda / 8 content slides / summary / contacts).
> content from the attached brief.docx.

That is style inheritance, not “editing the original.” Claude does not touch your template — it reads it and assembles new slides in that style.

Useful when:

  • You work in a corporation with a strict brand guide
  • A client sent a template and asks for “our look”
  • You have last year’s successful deck and need a new one in the same style

The deck as an artefact gets a full etape (06, pitch deck from .xlsx); here — only the template-upload technique.

Захват с живого сайта: стилевой и компонентный референсCapture from a live site: style & component reference

Помимо файлов, Claude Design принимает URL живого сайта как референс. Это сильнейший приём, когда вам нужен «как у Stripe.com», но точнее, чем эпитет.

> референс по стилю и компонентам: https://stripe.com
> собери лендинг для нашего инвест-сервиса в этой стилистике —
> крупная типографика, чистая сетка, продуктовые иллюстрации с градиентами,
> цветовые акценты на фоне нейтральной палитры.
> контент возьми из брифа.docx (приложен).

Claude захватывает стилевые сигналы: палитру, типографику, структуру композиции, паттерны интерактивных элементов. Не копирует код или конкретные графические элементы — перенимает дизайн-язык.

Хорошо работает для:

  • Промо-сайтов: Stripe, Linear, Vercel, Apple, Mercury, Notion
  • Журналов: Wallpaper, Monocle, NYT, FT, The New Yorker
  • Брендов: Aēsop, MUJI, Patagonia, Glossier
  • Прецедентов в нише: ваши конкуренты или вдохновители

Можно сослаться на несколько сайтов одновременно, описав что взять с каждого:

> стиль: типографика как у https://wallpaper.com (крупный serif),
> палитра как у https://aesop.com (приглушённая, тёплая),
> layout-сетка как у https://stripe.com.

Besides files, Claude Design accepts a live-site URL as a reference. The strongest move when you need “like Stripe.com” with more precision than an epithet.

> style and component reference: https://stripe.com
> build a landing for our investment service in this style —
> large typography, clean grid, product illustrations with gradients,
> colour accents on a neutral palette.
> content from the attached brief.docx.

Claude captures the style signals: palette, typography, composition, interaction patterns. It does not copy code or specific graphics — it inherits the design language.

Works well for:

  • Promo sites: Stripe, Linear, Vercel, Apple, Mercury, Notion
  • Magazines: Wallpaper, Monocle, NYT, FT, The New Yorker
  • Brands: Aēsop, MUJI, Patagonia, Glossier
  • Precedents in your niche: competitors or aspirational peers

You can cite multiple sites at once, naming what to take from each:

> style: typography like https://wallpaper.com (big serif),
> palette like https://aesop.com (muted, warm),
> layout grid like https://stripe.com.
Совмещение источников и границы загрузкиCombining sources & upload boundaries

Реальный мощный промпт почти всегда совмещает несколько источников:

> задача: лендинг для запуска инди-игры «Tide of Stars».
> контент: возьми из game-brief.docx (приложен) — описание, фичи, скриншоты.
> уровни: список из levels.xlsx (приложен) — секция «уровни».
> стиль: референсный сайт https://hadesgame.com.
> формат: 1280×800 десктоп + мобильная.
> структура: hero с trailer-видео, фичи в три колонки, уровни сеткой, dev-blog ссылка внизу.

Claude принимает все три источника одновременно: вытаскивает контент из .docx, повторяющиеся блоки из .xlsx, стилевой язык с URL.

Что НЕ переносится

  • PDF — импортируется только как текст. Layout PDF теряется. Если есть .docx-исходник — используйте его. Если только PDF — возможно, проще пересказать структуру в промпте.
  • SVG/.ai-исходники — иконки и иллюстрации Claude перерисовывает по своему пониманию. Полное соответствие исходнику не гарантируется. Если нужно использовать конкретную иконку как есть — экспортируйте её как PNG, и скажите «используй как есть».
  • Сложные Excel-формулы — Claude видит только итоговые значения, не формулу. Если важна сама формула (для дашборда расчётов) — вынесите её в отдельную колонку «формула как текст».
  • Защищённые/корпоративные сайты — URL-захват требует публичного доступа. Если ваш референс за паролем — экспортируйте скриншот.

Когда файл, когда промпт

Простое правило: если контент структурирован — файл; если идея или короткая инструкция — промпт.

  • Список из 3+ повторяющихся элементов → файл (.xlsx или .docx со списком).
  • Стилевые требования, которые меняются по ходу итераций → промпт.
  • Корпоративный шаблон → файл (.pptx).
  • Срочная правка одного блока → промпт.

A really powerful prompt almost always combines several sources:

> task: landing for an indie-game launch "Tide of Stars."
> content: from game-brief.docx (attached) — description, features, screenshots.
> levels: from levels.xlsx (attached) — "levels" sheet.
> style: reference site https://hadesgame.com.
> format: 1280×800 desktop + mobile.
> structure: hero with trailer-video, features in three columns, levels grid, dev-blog link at the bottom.

Claude consumes all three at once: content from .docx, repeating blocks from .xlsx, style language from the URL.

What does NOT carry over

  • PDF — imports as text only. PDF layout is lost. If a .docx source exists — use it. If only PDF — maybe easier to retell structure in the prompt.
  • SVG / .ai sources — Claude redraws icons and illustrations by its own reading. Exact source-match is not guaranteed. If you must use a specific icon as-is — export to PNG and say “use as-is.”
  • Complex Excel formulas — Claude sees only the resulting values, not the formula. If the formula itself matters (for a calculation dashboard) — put it in a “formula as text” column.
  • Protected / corporate sites — URL capture needs public access. If your reference is behind a login — export a screenshot.

File vs prompt

Simple rule: if content is structured — a file; if it’s an idea or short instruction — a prompt.

  • A list of 3+ repeating items → file (.xlsx or .docx with a list).
  • Style requirements that shift across iterations → prompt.
  • Corporate template → file (.pptx).
  • One-block urgent fix → prompt.
§ II · Практика по ходуPractice along the way

Один бриф, два путиOne brief, two paths

Этот короткий прогон ставит на место разницу между «текстовым промптом с брифом» и «загрузкой .docx». Заняет 10–15 минут.

A short run that locks in the difference between “a text prompt with the brief” and “a .docx upload.” 10–15 minutes.

Прогон · текстовый промпт vs .docxRun · text prompt vs .docx

  1. В Word/Google Docs создайте короткий бриф (1 страница) на любую тему: лендинг для собственного pet-project, услуги фрилансера, домашнего ивента. Структурируйте: h1 — название, h2 — три раздела (Цель, Аудитория, Тон), маркированный список ключевых требований.
  2. Скопируйте весь текст брифа. Откройте Claude Design, новый проект. Вставьте текст как обычный промпт. Дождитесь первой версии.
  3. Не уточняйте — запомните, что получилось.
  4. Сохраните бриф как .docx. Снова новый проект Claude Design. Прикрепите .docx в диалог. Промпт коротко: «собери лендинг по брифу выше, формат 1280×800, стиль возьми из раздела «Тон»».
  5. Сравните две первые версии. На что Claude обратил внимание во втором случае, чего не было в первом? Иерархия лучше? Структурные блоки точнее? Стиль ближе?
  1. In Word/Google Docs create a short brief (1 page) on any topic: a landing for your pet project, a freelance service, a home event. Structure it: h1 — name, h2 — three sections (Goal, Audience, Tone), bulleted list of key requirements.
  2. Copy the whole brief. Open Claude Design, new project. Paste as a regular prompt. Wait for the first take.
  3. Do not refine — remember the output.
  4. Save the brief as .docx. New Claude Design project again. Attach the .docx in the dialogue. Short prompt: “assemble the landing per the brief above, format 1280×800, style from the ‘Tone’ section.”
  5. Compare the two first takes. What did Claude pick up in the second one that wasn’t in the first? Sharper hierarchy? More accurate structural blocks? Closer style?

Что мы тренируем: чувствовать ROI загрузки. Структура документа становится структурой дизайна автоматически.

What we are training: feeling the ROI of upload. Document structure becomes design structure automatically.

Проект A · по инструкции · трек кофейняProject A · guided · café track

Меню кофейни из .xlsxCafé menu from .xlsx

Цель: меню A4 на печать, собранное из 9-строчной .xlsx-таблицы. Тренируем «таблица → повторяющиеся блоки».

Goal: an A4 print menu assembled from a 9-row .xlsx table. We train “table → repeating blocks.”

Шаги

Steps

  1. В Excel/Google Sheets создайте таблицу с колонками Раздел | Позиция | Описание | Цена и 9 строк (как в примере выше: 3 espresso + 3 filter + 3 sweets). Сохраните как .xlsx.
  2. Откройте Claude Design, новый проект. Прикрепите .xlsx.
  3. Промпт: 
    > меню кофейни «Северный край» из приложенной таблицы.
> формат A4 вертикальный, для печати.
> стиль: Müller-Brockmann (модульная сетка, sans-serif, минимум цвета, один акцент кирпично-красный).
> группировка: колонка «Раздел» — это секции (espresso / filter / sweets).
> в каждой секции — три позиции. покажи название (полужирный),

How Claude Design reads uploaded files: not as a wall of text but by parsing structure (headings, tables, lists, columns) and translating it into visual elements. An .xlsx becomes repeating blocks; a .docx brief becomes a brief for a landing.

§ VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

  1. В чём разница между «вставить бриф текстом в промпт» и «загрузить .docx с брифом»? What is the difference between “paste the brief as text” and “upload a .docx brief”?

    показать ответshow answer

    При вставке текстом теряется структура (заголовки, списки, таблицы). При загрузке .docx Claude парсит структуру и использует её как семантическую иерархию: h1 → заголовок страницы, h2 → секции, списки → блоки, таблицы → повторяющиеся элементы. Структурированный документ работает лучше длинного промпта.

    Pasting text loses structure (headings, lists, tables). With a .docx upload Claude parses the structure and uses it as semantic hierarchy: h1 → page title, h2 → sections, lists → blocks, tables → repeating elements. A structured document beats a long prompt.

  2. Как Claude использует .xlsx с 9-строчной таблицей меню кофейни? How does Claude use an .xlsx with a 9-row café menu table?

    показать ответshow answer

    Колонки становятся полями карточки (раздел, позиция, описание, цена), строки — повторами. Если в промпте указать «группировка по колонке Раздел» — получим три секции по три позиции. Обновили таблицу → перезагрузили в тот же диалог → дизайн обновился автоматически.

    Columns become card fields (section, item, description, price), rows become repeats. With “group by Section” in the prompt you get three sections of three items. Update the table → re-upload to the same dialogue → design updates automatically.

  3. Что Claude делает с загруженным .pptx-шаблоном — редактирует или генерирует новые слайды? What does Claude do with an uploaded .pptx template — edit or generate new slides?

    показать ответshow answer

    Генерирует новые. Это наследование стиля: исходный файл не трогается, Claude читает палитру, типографику, расположение логотипа, slide-master — и собирает новые слайды в этой стилистике. Полезно для работы с корпоративным брендгайдом.

    Generates new. This is style inheritance: the original file is untouched. Claude reads palette, typography, logo placement, slide-master — and assembles new slides in that style. Useful for corporate brand-guide work.

  4. Как захватить стиль с живого сайта-референса? How do you capture style from a live reference site?

    показать ответshow answer

    Дать URL прямо в промпте: «стиль и компоненты как у https://stripe.com». Claude захватывает дизайн-язык (палитру, типографику, layout-патт), не копируя код или конкретные графические элементы. Можно указать, что именно перенять: «типографику оттуда, layout оставь свой».

    Give a URL in the prompt: “style and components like https://stripe.com.” Claude captures the design language (palette, typography, layout patterns) without copying code or specific graphics. You can specify what to inherit: “typography from there, keep my own layout.”

  5. Что НЕ переносится корректно при загрузке файлов? What does NOT carry over cleanly when uploading files?

    показать ответshow answer

    PDF (теряет layout, импортируется как текст), SVG/.ai-исходники (Claude перерисовывает по своему пониманию), сложные Excel-формулы (видны только итоговые значения), сайты за паролем (URL-захват требует публичного доступа).

    PDF (loses layout, imports as text), SVG/.ai sources (Claude redraws from its own reading), complex Excel formulas (only computed values visible), password-gated sites (URL capture needs public access).

  6. Когда лучше загрузить файл, а когда написать всё в промпт? When upload, when prompt?

    показать ответshow answer

    Файл — если контент структурирован (3+ повторяющихся элемента, многораздельный документ, корпоративный шаблон). Промпт — для идеи, стилевых требований, срочных правок одного блока. Совмещение — норма: .docx + .xlsx + URL в одном диалоге работает лучше всего.

    File — if content is structured (3+ repeats, multi-section document, corporate template). Prompt — for ideas, style requirements, urgent one-block fixes. Combining is the norm: .docx + .xlsx + URL in one dialogue works best.

  7. Можно ли совместить .docx + .xlsx + URL в одном промпте? Can you combine .docx + .xlsx + URL in one prompt?

    показать ответshow answer

    Да, и это нормальный режим работы. Прикрепляете оба файла в диалог, в промпте указываете URL и объясняете, что откуда брать: «контент из брифа.docx, повторяющиеся блоки из levels.xlsx, стиль с https://...». Claude обрабатывает все три источника одновременно.

    Yes, and it’s the normal working mode. Attach both files in the dialogue, give the URL in the prompt, explain where each comes from: “content from brief.docx, repeating blocks from levels.xlsx, style from https://...”. Claude consumes all three at once.
