§ I · ТемыTopics

Из чего состоит хороший промптWhat makes a good prompt

Анатомия хорошего промпта: цель / ограничения / формат / примерAnatomy of a good prompt: goal / constraints / format / example

Сравните три способа попросить одно и то же:

Плохо:

> сделай калькулятор

Чуть лучше:

> сделай калькулятор сложного процента в html

Хорошо:

> сделай калькулятор сложного процента — одностраничник index.html.
> ввод: сумма, ставка %/год, срок в годах.
> вывод: итоговая сумма + таблица по годам + график.
> стиль — минималистичный, тёплая бумажная палитра, шрифт засечковый.
> одним файлом, без зависимостей, чтобы открывалось двойным кликом.

В хорошем промпте есть четыре слоя:

  1. Цель — что должно получиться. «калькулятор сложного процента».
  2. Ограничения — чего не делать или какими средствами. «одним файлом, без зависимостей».
  3. Формат — в каком виде получить результат. «одностраничник index.html», «ввод X, вывод Y».
  4. Пример или стиль — на что похоже. «минималистичный, тёплая бумажная палитра».

Чем больше ясности в этих четырёх слоях, тем точнее первая версия. Но не переусердствуйте: если каждый раз писать промпт на десять строк, вы потеряете в скорости. На простые задачи — короткий промпт.

Compare three ways to ask the same thing:

Bad:

> make a calculator

A bit better:

> make a compound interest calculator in html

Good:

> make a compound interest calculator — single-page index.html.
> inputs: principal, rate %/year, years.
> outputs: total + yearly table + chart.
> style: minimalist, warm paper palette, serif type.
> one file, no dependencies, opens with a double-click.

A good prompt has four layers:

  1. Goal — what should appear. “compound interest calculator”.
  2. Constraints — what not to do, what tools to use. “one file, no deps”.
  3. Format — how the result should look. “single-page index.html”, “inputs X, outputs Y”.
  4. Example or style — what it should resemble. “minimalist, warm paper palette”.

The clearer those four layers, the closer the first version. But do not overdo: writing a ten-line prompt for every trivial task kills your speed. Simple task — short prompt.

Когда давать длинный контекст, когда короткийWhen to give long context, when short

«Контекст» — это вся информация, которую Claude видит в текущем диалоге: ваши промпты, прикреплённые файлы, прошлые ответы. Чем больше контекста, тем больше Claude знает — но и тем дороже стоит каждый ответ.

Длинный контекст полезен, когда:

  • задача завязана на конкретные файлы/данные;
  • есть нетривиальные правила или стиль;
  • важна совместимость с уже сделанным.

Короткий контекст лучше, когда:

  • задача типовая;
  • вы не уверены, чего хотите — быстрее показать первую версию и уточнить;
  • вы только начинаете сессию.

Хорошее правило: сначала короткий промпт, потом доуточняем по ходу. Это быстрее, чем сразу писать большой.

“Context” is everything Claude sees in the current dialogue: your prompts, attached files, past replies. More context — more knowledge, but each reply costs more.

Long context helps when:

  • the task depends on specific files/data;
  • there are non-trivial rules or style;
  • compatibility with prior work matters.

Short context is better when:

  • the task is standard;
  • you are not sure what you want — show v1 fast, then refine;
  • you are just opening a new session.

Good rule: start with a short prompt, refine as you go. Faster than writing a big one upfront.

Итерация: «почти, но…», «нет, не так», «теперь добавь»Iteration: “almost, but…”, “no, not like that”, “now add”

Главный навык всего курса: не писать «идеальный» промпт сразу, а поправлять короткими шагами.

Полезные фразы для итерации:

  • «почти, но шрифт слишком крупный»;
  • «нет, я имел в виду…»;
  • «сделай ту же страницу, но в тёмной теме»;
  • «добавь кнопку «сбросить» рядом с «посчитать»»;
  • «сохрани текущую версию как v1.html и сделай v2.html, где…».

Claude быстрый. Пять-шесть итераций по 30 секунд работают лучше, чем десять минут шлифовки одного промпта.

The core habit of the whole course: don’t write a “perfect” prompt — refine in short steps.

Useful iteration phrases:

  • “almost, but the type is too big”;
  • “no, I meant…”;
  • “same page but in a dark theme”;
  • “add a ‘reset’ button next to ‘calculate’”;
  • “save the current as v1.html and make a v2.html where…”.

Claude is fast. Five-six iterations of thirty seconds beat ten minutes polishing one prompt.

Откат: «верни как было», «забудь последнее», EscRollback: “undo”, “forget that”, Esc

Если итерация увела куда-то не туда, нужно уметь возвращаться:

  • Esc — во время выполнения. Останавливает текущее действие, ничего не трогает.
  • «верни файл X как был до последнего изменения» — Claude использует git или внутренние backup.
  • «забудь то, что я только что сказал, начнём заново с вот этого…» — явно говорите, что прошлый промпт «не считается». Claude уберёт его из своих рассуждений.
  • «удали файл X» — самый прямой откат, если нужно начать с нуля.

Привыкайте откатываться без стеснения — это часть нормальной работы с агентом, а не признак ошибки.

If iteration drifted somewhere wrong, you need to roll back:

  • Esc — during execution. Stops the current action, touches nothing.
  • “revert file X to the state before the last change” — Claude uses git or its internal backup.
  • “forget what I just said, let’s start again from this…” — say explicitly that the previous prompt does not count. Claude removes it from its reasoning.
  • “delete file X” — the most direct rollback when you want to start clean.

Get comfortable rolling back — it is normal agent work, not a sign of failure.

Генерация вариантов: «сделай ещё две версии»Generating variants: “make two more versions”

Один из любимых приёмов: попросить Claude сделать несколько вариантов одной задачи. Полезно, когда вы не уверены, какой стиль/подход хотите.

> сохрани текущую версию как v1-classic.html.
> теперь сделай v2-minimal.html (более минималистичную)
> и v3-playful.html (с яркими цветами и анимацией).
> одни и те же данные, разный стиль.

На выходе — три файла. Открываете все три, сравниваете в браузере, выбираете любимый или просите что-то соединить.

Этот приём незаменим, когда:

  • вы ищете «свой» стиль и ещё не знаете его;
  • нужно показать заказчику/другу выбор;
  • хотите быстро увидеть, какие подходы Claude вообще считает уместными.

A favourite trick: ask Claude to produce several variants of one task. Useful when you are not sure which style/approach you want.

> save the current version as v1-classic.html.
> now make v2-minimal.html (more minimal)
> and v3-playful.html (bright colours, animations).
> same data, different styles.

You get three files. Open all in the browser, compare, pick a favourite or ask to merge.

This is irreplaceable when:

  • you are searching for “your” style and don’t know it yet;
  • you need options to show a client/friend;
  • you want a quick look at what approaches Claude considers appropriate.
Чтение результата: что Claude сделал, что предположил, что пропустилReading the result: what Claude did, assumed, missed

После каждого ответа Claude в терминале есть две части:

  1. Что он сделал — список изменений (созданные/изменённые файлы, выполненные команды). Это видно сразу.
  2. Что он предположил — решения, которые он принял за вас. Например: «использовал библиотеку Chart.js», «выбрал тёмную тему», «считал ставку годовой».

Привыкайте читать оба слоя. Если Claude предположил что-то не так — на следующей итерации поправьте: «нет, ставка должна быть месячной».

Если что-то пропустил — явно укажите: «ты не добавил кнопку сброса, добавь».

Это и есть «навык чтения результата» — один из ключевых для всего курса.

After every reply Claude has two parts:

  1. What it did — list of changes (files created/modified, commands run). Visible immediately.
  2. What it assumed — choices it made for you. E.g.: “used Chart.js,” “picked a dark theme,” “treated the rate as yearly.”

Get used to reading both layers. If Claude assumed something wrong — fix it next iteration: “no, the rate is monthly.”

If something is missing — say so: “you did not add a reset button, add it.”

This is the “result-reading skill,” one of the most important across the course.

Сохранение и возобновление сессий: --continue, --resumeSaving and resuming sessions: --continue, --resume

Сессия Claude Code не пропадает, когда вы закрываете терминал. Можно вернуться позже:

# продолжить последнюю сессию в текущей папке
$ claude --continue

# выбрать одну из недавних сессий из списка
$ claude --resume

Полезно, когда:

  • задача большая, вы вернулись на следующий день;
  • не хочется заново объяснять контекст;
  • был сбой и диалог закрылся неожиданно.

На этапе 04 разберём, как длинные сессии «уплотнять» через /compact, чтобы Claude не тонул в старых сообщениях.

A Claude Code session does not vanish when you close the terminal. You can come back:

# continue the latest session in the current folder
$ claude --continue

# pick one recent session from a list
$ claude --resume

Useful when:

  • the task is big, you returned the next day;
  • you don’t want to re-explain context;
  • a crash closed the dialogue unexpectedly.

On etape 04 we cover how to compact long sessions with /compact so Claude does not drown in old messages.

Прочитайте каждую пару и попробуйте сами сформулировать «хороший вариант» для своей задачи — например, для карточки фильма из этапа 00. Дальше будет прогон, где надо применить в диалоге.

Read each pair and try to phrase a “good version” for your own task — e.g. the film card from etape 00. Then a hands-on run where you apply this in dialogue.

Пять пар «плохой → хороший»Five “bad → good” pairs

1. Сайт

# плохо
> сделай мне сайт
# хорошо
> сделай одностраничник про мою кофейню «Утро».
> вверху имя и слоган, ниже меню (3 категории по 4 позиции),
> время работы, карта (просто адрес курсивом), футер с инстаграмом.
> стиль — как в скандинавском кафе, тёплые бежевые тона.

2. Калькулятор

# плохо
> калькулятор
# хорошо
> калькулятор калорий за день: ввод — продукты с весом,
> результат — общая калорийность + БЖУ.
> одна страница html, без зависимостей, простая таблица.

3. Игра

# плохо
> сделай игру
# хорошо
> сделай браузерную игру «угадай число от 1 до 100».
> компьютер загадывает, я угадываю, он подсказывает «больше/меньше».
> 7 попыток, в конце — статистика.
> одна html-страница, тёмная тема, крупные кнопки.

4. Анализ

# плохо
> проанализируй мои траты
# хорошо
> в файле expenses.csv — мои траты за квартал (дата, сумма, категория, заметка).
> собери short-отчёт в expenses-report.md:
>   1) общая сумма по месяцам;
>   2) топ-3 категорий;
>   3) две неожиданные заметки, которые требуют моего внимания.
> не давай советов «как экономить» — только факты.

5. Дизайн

# плохо
> сделай страницу красивой
# хорошо
> перепиши index.html в стиле японской газеты shimbun:
> засечковый шрифт, два столбца, ornate horizontal rules,
> никаких анимаций, бумажный фон.

1. Website

# bad
> make me a website
# good
> one-pager for my coffee shop "Morning."
> top — name and tagline, then menu (3 categories x 4 items),
> opening hours, map (just an italic address), footer with instagram.
> style — Scandinavian cafe, warm beige tones.

2. Calculator

# bad
> calculator
# good
> daily calorie calculator: input — foods with weight,
> output — total calories + protein/fat/carbs.
> one html page, no deps, simple table.

3. Game

# bad
> make a game
# good
> browser game "guess a number 1 to 100."
> computer picks, I guess, it hints "higher/lower."
> 7 attempts, statistics at the end.
> one html page, dark theme, large buttons.

4. Analysis

# bad
> analyse my expenses
# good
> expenses.csv contains my Q1 expenses (date, amount, category, note).
> produce a short report expenses-report.md:
>   1) totals by month;
>   2) top-3 categories;
>   3) two unusual notes that need my attention.
> don't suggest "how to save" — facts only.

5. Design

# bad
> make the page beautiful
# good
> rewrite index.html in Japanese shimbun style:
> serif type, two columns, ornate horizontal rules,
> no animations, paper background.

Что общего у «хороших»? Конкретное имя файла или сущности, измеримые ограничения («3 категории по 4 позиции», «7 попыток»), намёк на стиль через узнаваемое сравнение («скандинавское кафе», «японская газета shimbun»).

What good prompts share: a concrete file or entity name, measurable constraints (“3 cats × 4 items,” “7 attempts”), a style cue via recognisable comparison (“Scandinavian cafe,” “Japanese shimbun”).

Прогон · Одна страница, пять итерацийRun · One page, five iterations

  1. В папке claude-course создайте новую папку iter, запустите там claude.
  2. Промпт 1. «сделай простую страницу-таймер на 15 минут. кнопка старт/стоп, цифры крупным шрифтом, центр экрана».
  3. Откройте файл в браузере. Поправьте 2: «поменяй цвет цифр на тёплый оранжевый».
  4. Поправьте 3: «добавь кнопку «сброс» рядом со стартом, тех же размеров».
  5. Поправьте 4: «сохрани текущую версию как timer-classic.html. сделай ещё timer-noir.html — чёрный фон, неоновые цифры».
  6. Поправьте 5: «в noir добавь тиканье каждую секунду через ‹audio› (синтезируй через web-audio, без скачивания)».
  7. В конце сделайте промпт-«ревизию»: «покажи в виде списка, что ты предположил за меня, чего я не указал».
  1. Inside claude-course make a new folder iter, launch claude there.
  2. Prompt 1. “make a simple 15-minute timer page. start/stop button, large digits, centred”.
  3. Open the file in the browser. Fix 2: “change the digit colour to warm orange”.
  4. Fix 3: “add a ‘reset’ button next to start, same size”.
  5. Fix 4: “save current as timer-classic.html. now make a timer-noir.html — black bg, neon digits”.
  6. Fix 5: “in noir add a tick every second via ‹audio› (synthesise via web-audio, no downloads)”.
  7. Finally, an audit prompt: “list what you assumed for me that I did not specify”.

Что мы тренируем: работать через короткие шаги, генерировать варианты, видеть собственные пробелы в промптах через ревизию-вопрос.

What we train: working in short steps, generating variants, seeing your own gaps via the audit question.

Эти два проекта — «база» курса. На них учимся писать промпты, итерировать, генерировать варианты. К концу у вас два отдельных артефакта.

Проект A · по инструкцииProject A · guided

Калькулятор сложного процентаCompound interest calculator

Цель: одностраничник в браузере. Ввод — сумма, ставка, срок. Вывод — итог по годам в таблице и графиком.

Goal: a one-pager in the browser. Inputs — principal, rate, term. Outputs — yearly totals as a table and a chart.

Шаги

Steps

  1. Создайте папку compound в claude-course, запустите claude.
  2. Начальный промпт (используем все четыре слоя — цель, ограничения, формат, стиль):

§ V · Словарь этапаEtape glossary

Что добавилось к словарюNew entries in the glossary

Этап 01 расширяет понятия из этапа 00 и добавляет несколько новых. В тексте они подчёркнуты пунктиром — наведите/коснитесь, и всплывёт справка.

Etape 01 builds on etape 00 and adds a few new terms. They are underlined with dots in the text — hover or tap to see the tooltip.

ПромптPrompt
(расширено) Формулировка задачи. Хороший промпт состоит из четырёх слоёв: цель, ограничения, формат, пример / стиль. На простые задачи — короткий промпт, на сложные — все четыре слоя. (extended) The wording of a task. A good prompt has four layers: goal, constraints, format, example / style. Short prompt for simple tasks, all four for complex ones.
КонтекстContext
Вся информация, которую Claude учитывает в текущем диалоге: ваши промпты, прикреплённые файлы, прошлые ответы. Чем длиннее контекст — тем больше Claude знает, тем дороже стоит каждый ответ. На этапе 04 разбираем, как им управлять. All information Claude considers in the current dialogue: your prompts, attached files, past replies. Longer context — more knowledge, higher cost per reply. Etape 04 covers how to manage it.
ИтерацияIteration
(повтор) Повторный заход по той же задаче с поправкой. Главный навык курса — не писать «идеальный» промпт сразу, а поправлять короткими шагами. (recap) Going through the same task again with a correction. The main course skill — don’t write a “perfect” prompt, refine in short steps.
ОткатRollback
Команда вернуть Claude или файл к предыдущему состоянию. Способы: Esc (во время выполнения), «верни как было», «забудь предыдущее», «удали этот файл». A command to return Claude or a file to a previous state. Forms: Esc during execution, “revert,” “forget the previous,” “delete this file.”
ВариантыVariants
Несколько разных версий одной задачи, сгенерированных одним запросом. «сохрани как v1, теперь сделай v2 в другом стиле». Полезно, когда не уверены, какой подход хотите. Several different versions of one task, generated in one request. “save as v1, now make a v2 in another style.” Useful when you do not know which approach you want.
СессияSession
Один диалог с Claude от запуска до выхода. Сессия не пропадает при закрытии терминала — продолжите через claude --continue или выберите из списка через claude --resume. One dialogue with Claude from launch to exit. A session does not vanish when you close the terminal — resume via claude --continue or pick from a list via claude --resume.
Аудит-промптAudit prompt
Финальный вопрос диалога: «перечисли, что ты принял за меня — цвета, формулу, валюту, шрифты». Помогает увидеть «слепые пятна» вашего промпта и сделать их явными в следующий раз. A final question in a dialogue: “list what you assumed for me — colours, formula, currency, fonts.” Helps surface blind spots in your prompt and make them explicit next time.
CDNCDN
Сервер, с которого браузер скачивает готовую библиотеку (например, chart.js) при открытии страницы. В промпте «через CDN» означает: не качай и не ставь — вставь ссылку, и всё. A server the browser downloads a ready-made library from (e.g. chart.js) when opening the page. “Via CDN” in a prompt means: do not download / install — just include a URL.
localStoragelocalStorage
«Память» браузера для одной страницы — данные сохраняются между перезагрузками. Используем для рекордов, сохранённых тем, состояния игр. A browser’s “memory” for one page — data persists across reloads. Used for highscores, saved themes, game state.
brainstormingbrainstorming
Skill, который учит Claude задавать уточняющие вопросы перед началом задачи. Полезен, когда вы только начинаете и не всегда сами знаете, чего хотите. Установлен опционально в § 0. § VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

Постарайтесь сначала ответить мысленно или вслух — только потом раскрывайте ответ. После сравнения отметьте: знал или не знал. По итогам подскажу, что повторить.

  1. Из каких четырёх слоёв состоит хороший промпт?What are the four layers of a good prompt?

    показать ответshow answer

    Цель (что должно получиться) · ограничения (чего не делать / какими средствами) · формат (в каком виде получить результат) · пример / стиль (на что похоже).

    Goal (what should appear) · constraints (what not to do / which tools) · format (how the result should look) · example / style (what it should resemble).

  2. Что лучше: написать «идеальный» десяти-строчный промпт сразу или начать с короткого и поправлять? Почему?Better: write a “perfect” ten-line prompt up front, or start short and refine? Why?

    показать ответshow answer

    Лучше короткий + итерации. Claude быстрый: 5–6 поправок по 30 секунд работают точнее и быстрее, чем 10 минут шлифовки одного промпта. Кроме того, вы увидите, что Claude предположил — это сразу подсветит ваши собственные пробелы.

    Short + iterations. Claude is fast: 5–6 thirty-second tweaks beat ten minutes polishing one prompt. Plus you see what Claude assumed — that immediately surfaces your own blind spots.

  3. Чем отличается «откат» через Esc от «верни файл как был»?How does Esc rollback differ from “revert the file”?

    показать ответshow answer

    Esc — останавливает текущее действие во время выполнения, ничего не трогает. «верни файл как был» — откатывает уже сделанные изменения через git/backup. Первое — «не делай вот это», второе — «отмени то, что сделал».

    Esc stops the current action mid-execution, touching nothing. “revert the file” rolls back already-made changes via git/backup. First is “don’t do this,” second is “undo what you did.”

  4. Зачем просить Claude сгенерировать несколько вариантов одной страницы? Когда это особенно полезно?Why ask Claude to generate several variants of one page? When is it especially useful?

    When you do not know which style/approach you want. Comparing three open tabs in the browser is easier than describing styles in words. Also handy when showing options to a client/friend — and to see at a glance what approaches Claude even considers fitting.

  5. Что такое «аудит-промпт» и зачем он нужен?What is an “audit prompt” and what is it for?

    показать ответshow answer

    Финальный вопрос: «перечисли, что ты принял за меня — цвета, формулу, валюту, шрифты». Подсвечивает «слепые пятна» промпта — места, где Claude сделал выбор за вас. В следующий раз сможете указать эти параметры явно или сознательно отдать на его выбор.

    A final question: “list what you assumed for me — colours, formula, currency, fonts.” Surfaces blind spots — the choices Claude made for you. Next time you can either set those explicitly or knowingly delegate them.

  6. Как продолжить вчерашнюю сессию Claude Code в той же папке, не объясняя контекст заново?How do you resume yesterday’s Claude Code session in the same folder without re-explaining context?

    показать ответshow answer

    claude --continue — продолжит последнюю сессию в текущей папке. claude --resume — покажет список недавних сессий, можно выбрать любую.

    claude --continue resumes the latest session in the current folder. claude --resume lists recent sessions and lets you pick any.

  7. Промпт «сделай страницу красивой» — почему это плохо? Как переписать?The prompt “make the page beautiful” — why is it bad? How to rewrite?

    показать ответshow answer

    “Beautiful” is abstract — everyone defines it differently. Claude will pick a “mid-AI default” that probably won’t match yours. Better: give a recognisable comparison (Japanese newspaper, Scandinavian cafe, brutalism, Bauhaus) and concrete parameters (typeface, palette, width, spacing).
