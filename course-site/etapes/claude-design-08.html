Что мы пройдёмWhat we will walk through

3D-тур: стилизованная сцена с переключаемыми материалами3D tour: a stylised scene with switchable materials

Самая «волшебная» возможность для трека «квартира» (и для архитектурных продуктов вообще) — стилизованный 3D-тур. Вы даёте план комнат и список вариантов отделки, Claude Design генерирует кликабельную 3D-сцену.

> 3D-тур по квартире-студии 45 м². план приложен (apartment.xlsx, лист «rooms»).
> стиль: scandinavian modern, тёплая палитра, дневной свет.
> для каждой комнаты — 4 ракурса (входной угол, центр, окно, потолок).
> интерактив: переключатель материалов (3 варианта пола: дуб светлый / дуб тёмный / бетон),
>   переключатель стен (белый / шалфей / тёмно-синий),
>   переключатель освещения (день / закат / ночь).
> формат: широкий 1920×1080 для презентации.
> экспорт: HTML с интерактивностью (для отправки клиенту по ссылке).

Claude собирает стилизованную 3D-сцену — не фотореализм Corona/V-Ray, но осмысленная композиция с узнаваемой геометрией мебели и правильным светом. Достаточно, чтобы клиент за 15 секунд понял «сюда мы идём».

Что точно работает: переключение цветов и материалов, изменение освещения, перемещение камеры между фиксированными ракурсами.

Что нужно делать вручную/в Corona: фотореалистичный финальный рендер для рекламы или для просмотра в 1:1 масштабе на большом экране.

The most “magical” capability for the apartment track (and architectural products in general) — a stylised 3D tour. You give a floor plan and a list of finishes, Claude Design generates a clickable 3D scene.

> 3D tour of a 45 m² studio apartment. plan attached (apartment.xlsx, "rooms" sheet).
> style: scandinavian modern, warm palette, daylight.
> per room — 4 angles (entry corner, centre, window, ceiling).
> interactive: material switcher (3 floors: light oak / dark oak / concrete),
>   wall switcher (white / sage / deep blue),
>   lighting switcher (day / sunset / night).
> format: wide 1920×1080 for presentation.
> export: HTML with interactivity (for sending the client a link).

Claude assembles a stylised 3D scene — not photoreal Corona/V-Ray, but a sensible composition with recognisable furniture geometry and correct light. Enough for a client to grasp “this is where we’re going” in 15 seconds.

What works reliably: switching colours/materials, changing lighting, moving camera across fixed angles.

What still needs Corona: a photoreal final render for ads or 1:1 viewing on a big screen.

Shader-фон: «нескучный» hero без After EffectsShader background: a non-boring hero without After Effects

Shader — это процедурная графика, считающаяся в браузере на GPU. То, что у вас постоянно перед глазами на Linear, Stripe, OpenAI — чуть-чуть мерцающие градиенты, плавающие пятна света, переливы. Раньше это требовало WebGL и знания GLSL. Сейчас — одного промпта.

> shader-фон для hero-секции лендинга «Tide of Stars».
> визуально: глубокое звёздное небо с медленными переливами синего → фиолетового,
>   мягкие пятна света, легкое смещение от движения мыши.
> перформанс: 60fps на среднем mac, должно работать на mobile.
> код: production-ready, передаётся в handoff bundle для Claude Code.

На холсте появится hero с живым шейдер-фоном. Можно дальше править через диалог: «медленнее переливы», «добавь падающую звезду каждые 10 секунд», «уменьши контраст синего».

Когда нужен shader:

  • Hero лендинга, который должен «дышать» (Stripe, Linear, OpenAI-тип сайтов).
  • Атмосфера в игровом UI (этап 04).
  • Лоадер для долгих операций.

Когда не нужен:

  • Контентная страница (читать текст). Shader отвлекает.
  • Печать — shader не распечатается.
  • Слабые устройства — в handoff Claude Code добавит fallback на статичный градиент.

A shader is procedural graphics computed in the browser on the GPU. The thing you see on Linear, Stripe, OpenAI — gently shifting gradients, drifting light blobs. Used to require WebGL + GLSL. Now it’s one prompt.

> shader background for the "Tide of Stars" landing hero.
> visual: deep starlit sky with slow blue → violet shifts,
>   soft light blobs, mild displacement on mouse movement.
> perf: 60fps on mid-range mac, must work on mobile.
> code: production-ready, ships in the handoff bundle to Claude Code.

The canvas shows a hero with a live shader background. Refine via dialogue: “slower shifts,” “add a shooting star every 10s,” “reduce blue contrast.”

When you need a shader:

  • Landing hero that must “breathe” (Stripe, Linear, OpenAI-type sites).
  • Atmosphere in game UI (etape 4).
  • Loader for long ops.

When you don’t:

  • Content page (text-reading). Shader distracts.
  • Print — shaders don’t print.
  • Weak devices — on handoff Claude Code adds a static-gradient fallback.
Voice UI: микрофон в интерфейсе AI-консультантаVoice UI: a microphone in the AI advisor interface

Голосовой ввод — одна из тех штук, что Figma не умеет вообще. Claude Design генерирует базовый voice-aware UI: микрофон-кнопка, индикатор записи, визуализация уровня звука, transcript-stream справа. Не production-grade, но для прототипа AI-консультанта или voice-search — достаточно.

> интерфейс AI-консультанта для инвест-кабинета.
> центральная зона: чат с пользователем (как Claude.ai, в стиле workspace).
> внизу — кнопка микрофона (большой круг), индикатор записи (animated waveform),
>   transcript появляется в реальном времени над инпутом.
> справа панель — список «спросите меня про»: 5 типичных вопросов.
> формат: 1280×800 десктоп.
> voice API: Web Speech (Whisper не нужен для прототипа).

Получите кликабельный прототип, где кнопка микрофона реально записывает в браузере (Web Speech API), показывает транскрипт. На этом этапе Claude обработка ответа — имитация (хардкод). Реальная интеграция с Claude API делается в Claude Code (этап 06 основного курса).

Voice input is one thing Figma can’t do at all. Claude Design generates a basic voice-aware UI: mic button, recording indicator, audio-level visualisation, transcript stream on the right. Not production-grade, but enough for a voice-advisor or voice-search prototype.

> AI advisor UI for the investment-office.
> centre: chat with user (like Claude.ai, in workspace style).
> bottom — mic button (large circle), recording indicator (animated waveform),
>   transcript appears live above the input.
> right panel — "ask me about": 5 typical questions.
> format: 1280×800 desktop.
> voice API: Web Speech (no Whisper needed for the prototype).

You get a clickable prototype where the mic actually records in the browser (Web Speech API) and shows the transcript. At this etape Claude’s response handling is mocked. Real Claude API integration happens in Claude Code (main course etape 06).

Motion-переходы между сценариямиMotion transitions between scenarios

Анимация перехода между состояниями — маленькая, но «дорогая» деталь, отличающая premium-продукт от дешёвого. В статичном дизайне её нет, в After Effects она требует часа работы, в Claude Design — одна фраза.

> на лендинге «Личного инвест-кабинета» есть переключатель «сценарий прогноза»:
>   pessimistic / base / optimistic.
> добавь анимацию перехода между сценариями:
>   — числа в карточках плавно меняются (counting-animation),
>   — линия графика morph-анимацией перерисовывается из старого состояния в новое,
>   — donut-chart distribuции тоже плавно изменяет сегменты,
>   — фон чуть-чуть меняет тёмность (тревожнее для pessimistic, теплее для optimistic).
> общая длительность 600–800ms, easing — cubic-bezier(0.2, 0.7, 0.2, 1).

Получаете живой переход. Для финального деплоя через handoff Claude Code оптимизирует анимации (CSS view-transitions, GPU-accelerated transforms). Но на этапе прототипа уже видно, как продукт «дышит».

State transition animation — a small but “expensive” detail separating premium from cheap. Absent in static design, an hour in After Effects, one sentence in Claude Design.

> on the "Personal Investment Office" landing there’s a "forecast scenario" switcher:
>   pessimistic / base / optimistic.
> add transition animation between scenarios:
>   — card numbers count smoothly (counting animation),
>   — chart line morphs from old to new shape,
>   — donut segments smoothly resize,
>   — background slightly shifts darkness (anxious for pessimistic, warmer for optimistic).
> total 600–800ms, easing cubic-bezier(0.2, 0.7, 0.2, 1).

Live transition. On final handoff, Claude Code optimises (CSS view-transitions, GPU-accelerated transforms). But on the prototype stage you already see the product “breathing.”

Реалистичные ожидания: что точно НЕ умеетRealistic expectations: what it does NOT do

Перечислю явно, чтобы не было разочарований:

  • Фотореалистичный 3D-рендер — уровень Corona, V-Ray, Octane. Claude Design делает стилизованный 3D, не photoreal. Для рекламы недвижимости в ЦИАНе — Corona.
  • Видео-монтаж — нарезка, наложение, цветокоррекция. Это работа Premiere/DaVinci. Claude Design может встроить уже готовое видео в дизайн.
  • 3D-моделирование персонажей — Blender, ZBrush. Claude может использовать готовую модель как референс для стилизации.
  • Сложная музыка/звуковой дизайн — Ableton, Logic. Claude может вставить готовый аудио-файл.
  • Production-grade voice UI — интеграция с OpenAI Whisper, Claude API, нормальная обработка ошибок, fallback на текст. Это работа Claude Code.

Главное правило: Claude Design делает прототип, а не продакшен. Если кому-то нужно посмотреть и понять идею — достаточно. Если запускаете в продакшен — передавайте через handoff в Claude Code, оттуда в нужные специальные инструменты.

Listed explicitly to avoid disappointment:

  • Photoreal 3D rendering — Corona, V-Ray, Octane. Claude Design does stylised 3D, not photoreal. For real-estate ads — Corona.
  • Video editing — cuts, overlays, grading. Premiere/DaVinci. Claude Design can embed a finished video in the design.
  • 3D character modelling — Blender, ZBrush. Claude can use a finished model as reference for stylisation.
  • Complex music / sound design — Ableton, Logic. Claude can drop in an audio file.
  • Production-grade voice UI — OpenAI Whisper, Claude API integration, error handling, text fallback. That’s Claude Code.

Headline rule: Claude Design ships prototypes, not production. If someone needs to see and grasp an idea — enough. For production — hand off to Claude Code, then to specialist tools.

§ II · Практика по ходуPractice along the way

Один shader за 15 минутOne shader in 15 min

Прогон · shader-hero для существующего лендингаRun · shader hero for an existing landing

  1. Возьмите любой свой лендинг (этап 01 или 02). Откройте.
  2. Промпт: «в hero-секции замени статичный фон на shader: медленные переливы из двух цветов workspace, лёгкий drift, 60fps».
  3. Дождитесь. Откройте Internal URL, посмотрите на живой фон.
  4. Уточните слайдером «speed» — найдите момент, когда фон «дышит», а не «мельтешит».
  5. Экспорт: handoff bundle. Откройте в Claude Code: «проверь shader-перформанс на mobile, добавь fallback на статичный градиент для prefers-reduced-motion».
  1. Take any existing landing (etape 1 or 2). Open it.
  2. Prompt: “in the hero replace the static background with a shader: slow shifts of two workspace colours, light drift, 60fps.”
  3. Wait. Open the Internal URL, watch the live background.
  4. Tune via slider “speed” — find the “breathing” sweet spot, not “flickering.”
  5. Export handoff bundle. In Claude Code: “check mobile shader perf, add static-gradient fallback for prefers-reduced-motion.”

§ VI · Quiz

Семь вопросовSeven questions

  1. Чем «стилизованный 3D-тур» отличается от Corona-рендера?Stylised 3D tour vs Corona render?

    show

    Стилизованный — графический, узнаваемая геометрия и правильный свет, без photoreal-материалов. Corona — фотореализм 1:1 с real-world. Claude Design делает первое — этого хватает для презентации клиенту, но не для рекламы недвижимости.

    Stylised is graphic, recognisable geometry and correct light, no photoreal materials. Corona is 1:1 photorealism. Claude Design does the first — enough for client pitch, not for real-estate ads.

  2. Когда нужен shader-фон, а когда нет?When shader, when not?

    show

    Нужен: hero лендинга, который должен «дышать» (Stripe/Linear-стиль), атмосфера в игровом UI, лоадер. Не нужен: контентные страницы (отвлекает), печать, слабые устройства (хотя есть fallback).

    Yes: landing hero that must “breathe” (Stripe/Linear-style), game-UI atmosphere, loader. No: content pages (distracts), print, weak devices (though fallback exists).

  3. Что Claude Design генерирует для voice UI на этапе прототипа?Voice UI in prototype stage?

    show

    Базу через Web Speech API: кнопка микрофона, индикатор записи, audio-визуализация, real-time transcript. Без интеграции с Whisper/Claude API — это работа Claude Code в handoff.

    The basics via Web Speech API: mic button, recording indicator, audio visualisation, live transcript. Without Whisper/Claude API integration — that’s Claude Code work post-handoff.

  4. Какие 3 инструмента переключаемые в 3D-туре?3 switchable tools in a 3D tour?

    show

    Материалы (пол/стены), освещение (день/закат/ночь), ракурс камеры. Это надёжно работает. Дополнительные эффекты (туман, погода) могут быть капризны.

    Materials (floor/walls), lighting (day/sunset/night), camera angle. Reliable. Extra effects (fog, weather) can be flaky.

  5. Что Claude Design точно НЕ умеет?What does Claude Design NOT do?

    show

    cubic-bezier(0.2, 0.7, 0.2, 1) — quick start, soft slow-down. 600–800ms total — typical “premium” range.
