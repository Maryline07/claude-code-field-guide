Claude Design vs Claude Code: что общее, что разноеClaude Design vs Claude Code: what is shared, what is new

Claude Design — это сиблинг Claude Code, заточенный под визуальные артефакты вместо файлов. Большая часть навыков из основного курса переносится напрямую. Меняется среда, инструменты обратной связи и формат вывода.

Что общее:

  • Та же модель — Opus 4.7 (теперь vision-modus в явном виде: Claude видит картинки).
  • Та же подписка — Pro/Max/Team/Enterprise.
  • Та же логика «арт-директор → исполнитель», итерация по диалогу, конкретные референсы лучше эпитетов.
  • Тот же приём «спросите Claude напоследок» — работает один в один.

Что меняется:

  • Среда. Не терминал, а браузер на claude.ai/design. Workspace вместо CLI.
  • Медиум. Не файлы, а холст с пространственной композицией. Это требует нового навыка — «читать дизайн» (отдельная тема ниже).
  • Три инструмента уточнения вместо одного. В Claude Code всё через текстовый промпт. Здесь: текстовый промпт, инлайн-комментарий на конкретном элементе холста, слайдеры для тонкой подстройки. Разные инструменты для разных задач — разбираем в этапе 01.
  • Native экспорт в дизайн-форматы. PDF, PPTX, Canva, HTML, folder, Internal URL. Без скриптов, одной кнопкой.
  • Persistent history. Все итерации сохраняются автоматически. В Claude Code это даёт git, тут — встроено в workspace.
  • Handoff в Claude Code. Дизайн упаковывается в bundle и уходит напрямую в Claude Code на имплементацию. Цикл «идея → дизайн → код → деплой» закрывается двумя инструментами без ручной интеграции — разбираем в этапе 05.

Запущен 17 апреля 2026 года, на момент прохождения курса в research preview — UI может отличаться от скриншотов в этом тьюториале. Принципы — нет.

Claude Design is Claude Code’s sibling, scoped to visual artefacts instead of files. Most of your main-course skills port over directly. What changes: the environment, the feedback tools, the output format.

What is shared:

  • The same model — Opus 4.7 (now in explicit vision mode: Claude sees images).
  • The same subscriptions — Pro / Max / Team / Enterprise.
  • The same “art director → executor” logic, dialogue-driven iteration, concrete references over epithets.
  • The “ask Claude last” move — works identically.

What changes:

  • Environment. Not a terminal, a browser at claude.ai/design. A workspace instead of a CLI.
  • Medium. Not files, a canvas with spatial composition. Requires a new skill — “reading a design” (its own topic below).
  • Three refinement tools instead of one. In Claude Code everything is the text prompt. Here: text prompt, inline comment on a specific canvas element, sliders for fine tuning. Different tools for different jobs — etape 01 covers boundaries.
  • Native export to design formats. PDF, PPTX, Canva, HTML, folder, Internal URL. No scripts, one click.
  • Persistent history. Every iteration saved automatically. Where Claude Code uses git, this is built in.
  • Handoff to Claude Code. The design packs into a bundle and ships straight to Claude Code for implementation. The “idea → design → code → deploy” loop closes across two tools without manual integration — covered in etape 05.

Launched 17 April 2026, currently in research preview — the UI may differ from the screenshots in this tutorial. Principles will not.

Подписка и Enterprise-доступ — короткоSubscription and Enterprise access — in brief

Подписка та же, что для основного курса. Доступ есть на Pro, Max, Team, Enterprise. Если вы уже подписаны — ничего делать не нужно, claude.ai/design просто откроется.

Одна особенность для Enterprise. В корпоративных организациях Claude Design выключен по умолчанию. Это сделано специально — политики конфиденциальности могут требовать ревью до выдачи инструмента сотрудникам.

Если вы Enterprise-юзер и claude.ai/design отвечает «not available»:

  1. Если вы админ организации — Settings → Labs / Beta features → Claude Design, тумблер вкл. Подождите пару минут.
  2. Если не админ — пишите в IT/HR со ссылкой на анонс продукта. Anthropic явно подтверждает: дизайны не идут на тренировку моделей и соблюдают ту же политику, что остальной Claude.

На этап 00 хватит базовых лимитов даже Pro. Если упрётесь в потолок — либо ждите окно, либо включите opt-in pay-as-you-go.

The subscription is the same as for the main course. Access on Pro, Max, Team, Enterprise. If you are already subscribed — do nothing, claude.ai/design simply opens.

One Enterprise quirk. In corporate orgs Claude Design is off by default — intentional, since confidentiality policies may need a review before staff get the tool.

If you are an Enterprise user and claude.ai/design says “not available”:

  1. If you are the org admin — Settings → Labs / Beta features → Claude Design, flip the toggle. Wait a couple of minutes.
  2. If not — ping IT/HR with a link to the launch post. Anthropic explicitly confirms: designs do not go into model training and follow the same policy as the rest of Claude.

Etape 00 fits inside Pro’s baseline allowance. If you hit a ceiling — wait for the window or enable opt-in pay-as-you-go.

Первый запуск: разбираем интерфейс claude.ai/designFirst launch: getting oriented at claude.ai/design

Открываем claude.ai/design. Перед вами три зоны:

  1. Слева — диалог. Сюда пишете промпты, прикрепляете файлы (.docx, .pptx, .xlsx, картинки), вставляете ссылки. Это «вход».
  2. В центре — холст. Здесь живёт сам дизайн. Размер холста подстраивается под формат: A4 для печати, 1080×1080 для соцсетей, 1920×1080 для презентации.
  3. Сверху — меню. Тут переключатель вида (Design / Code), кнопка Share (доступ для других), Export (PDF, PPTX, Canva, HTML, folder, ссылка), Handoff to Claude Code (для разработчика).

Слева сверху — ваши прошлые проекты («History»). Каждый проект — это отдельный диалог со своим холстом и историей правок.

Если что-то не нравится — просто закройте проект и начните новый. Claude Design ничего не ломает на вашем компьютере; всё живёт внутри браузера.

Open claude.ai/design. You see three zones:

  1. Left — the dialogue. Where you write prompts, attach files (.docx, .pptx, .xlsx, images), drop links. The “input.”
  2. Centre — the canvas. Where the design lives. Canvas size adapts to format: A4 for print, 1080×1080 for social, 1920×1080 for a slide.
  3. Top — the menu. Here: view switch (Design / Code), Share (access for others), Export (PDF, PPTX, Canva, HTML, folder, link), Handoff to Claude Code (for developers).

Top-left — your past projects (“History”). Each project is a separate dialogue with its own canvas and edit history.

If something feels off — just close the project and start a new one. Claude Design does not touch your machine; everything lives inside the browser.

Первый артефакт за тридцать минутA first artefact in thirty minutes

Самый быстрый способ почувствовать продукт — собрать обложку. Структура промпта вам знакома (цель, формат, стиль, структура), новое — где вы это пишете и чем уточняете.

# «New project» → пишем в диалог
> обложка-меню для крафтовой кофейни «Северный край».
> формат A4 вертикальный, для печати.
> стиль: скандинавская типографика — Müller-Brockmann meets Acne Studios.
> структура: вверху крупный заголовок MENU, под ним — название кофейни;
> в центре — три раздела (espresso / filter / sweets) с тремя позициями в каждом;
> внизу — адрес-заглушка и часы.

Первая версия за 30–60 секунд. Дальше — пробуем все три инструмента уточнения, чтобы почувствовать их на вкус (детально границы — в этапе 01):

  • Текстовый промпт в диалог — для смены концепции или замены раздела. Пример: «поменяй стиль на editorial, как у Wallpaper».
  • Инлайн-комментарий — замечание, привязанное к конкретному элементу на холсте. Жест выбора элемента может называться по-разному (наведение, клик, контекстное меню) — UI в research preview меняется. Принцип постоянный: привязать заметку к конкретной штуке, чтобы Claude однозначно понял, о чём речь.
  • Слайдер — для тонкой подстройки spacing, color, layout без слов. Тянете полоску, смотрите холст в реальном времени, ловите момент.

Когда нравится — сверху Export → PDF. Файл в Downloads. Это и есть ваш первый артефакт в Claude Design.

The fastest way to feel the product is to assemble a cover. The prompt anatomy is familiar (goal, format, style, structure); what is new is where you write it and how you refine.

# «New project» → into the dialogue
> menu cover for a craft café "Northern Coast."
> format A4 vertical, for print.
> style: Scandinavian typography — Müller-Brockmann meets Acne Studios.
> structure: top — large MENU heading, below — café name;
> centre — three sections (espresso / filter / sweets), three items each;
> bottom — placeholder address and hours.

First take in 30–60 seconds. Then — try all three refinement tools to taste them (etape 01 covers boundaries in detail):

  • Text prompt in the dialogue — for concept swap or section replacement. Example: “change the style to editorial, à la Wallpaper.”
  • Inline comment — a note attached to a specific element on the canvas. The element-select gesture may vary (hover, click, context menu) — research-preview UI shifts. The principle stays: attach a note to a specific thing, so Claude knows unambiguously what you mean.
  • Slider — wordless fine-tuning of spacing, colour, layout. Drag, watch the canvas live, catch the moment.

When you like it — top bar, Export → PDF. File in Downloads. That is your first Claude Design artefact.

Чтение дизайна: три слоя визуального чтенияReading a design: three layers

В Claude Code feedback бинарен: тесты зелёные или красные, ESLint молчит или ругается. В дизайне feedback неоднозначен — никто не скажет, что лендинг «сломан». Но вы всё равно должны уметь сформулировать, что именно «не так», чтобы Claude мог поправить. Это и есть главный новый навык — визуальная грамотность, которой основной курс не учил.

Помогает читать дизайн в трёх слоях. Каждый слой — отдельный канал диагностики, и каждый чинится своим инструментом уточнения.

Слой 1. Композиция: что где.

  • Распределение элементов по холсту: где hero, где блоки контента, где CTA, где футер.
  • Центр композиции — куда падает глаз в первую секунду.
  • Баланс: уравновешен ли холст, или один блок «тяжелее» соседних.

Если что-то «не на своём месте» — это композиция. Чините инлайн-комментарием на конкретном элементе.

Слой 2. Иерархия: что главное.

  • В каком порядке глаз проходит по странице. Что прочитывается первым? Вторым?
  • Цена видна раньше или позже описания? Имя бренда читается раньше или позже слогана?
  • Размер, вес, цвет, контраст — всё, что выделяет «главное» из «второстепенного».

Если самое важное теряется в шуме — это иерархия. Чините текстовым промптом: «выдели цену крупнее», «слоган подзаголовком, не равным h1».

Слой 3. Атмосфера: какой mood.

  • Тёплый / холодный, плотный / воздушный, дорогой / доступный, формальный / игривый.
  • Какая аудитория зайдёт на этот лендинг и купит?
  • На каком «соседе» этот дизайн стоит бы в воображаемом ряду — рядом с Apple, рядом с Etsy, рядом с Ray Gun?

Если общий mood мимо — это атмосфера. Чините слайдером: warmth, contrast, density.

Главная польза трёх слоёв — не «три категории», а прямая связь между диагнозом и инструментом. Композиция → инлайн. Иерархия → текст. Атмосфера → слайдер. Запомните это правило — и все ваши последующие правки станут точнее.

In Claude Code, feedback is binary: tests green or red, ESLint quiet or shouting. In design, feedback is ambiguous — no one will tell you the landing is “broken.” You still have to articulate what specifically is “off” so Claude can fix it. That is the new skill — visual literacy — the main course did not teach.

Three layers help you read. Each is a separate diagnostic channel, and each fixes with its own refinement tool.

Layer 1. Composition: what goes where.

  • Element distribution: where the hero, content blocks, CTA, footer sit.
  • Compositional centre — where the eye lands in the first second.
  • Balance: is the canvas weighted evenly, or is one block heavier than its neighbours.

If something is “in the wrong place” — that is composition. Fix it with an inline comment on the specific element.

Layer 2. Hierarchy: what matters most.

  • The order in which the eye traverses the page. What reads first? Second?
  • Is price visible before or after the description? The brand name before or after the slogan?
  • Size, weight, colour, contrast — whatever lifts “important” out of “secondary.”

If the important thing gets lost in the noise — that is hierarchy. Fix it by text prompt: “promote the price,” “slogan as a subheading, not equal to h1.”

Layer 3. Atmosphere: what mood.

  • Warm vs cool, dense vs airy, expensive vs accessible, formal vs playful.
  • What audience will land on this and convert?
  • In an imaginary line-up, who is this design’s neighbour — Apple, Etsy, Ray Gun?

If the overall mood is off — that is atmosphere. Fix it with a slider: warmth, contrast, density.

The headline value of the three layers is not “three categories,” it is a direct mapping from diagnosis to tool. Composition → inline. Hierarchy → text. Atmosphere → slider. Memorise that rule — every future correction gets sharper.
Сохранение, экспорт, доступSaving, exporting, sharing

В Claude Design нет кнопки «Save». Каждая ваша правка автоматически фиксируется в истории проекта. Закрыли вкладку — вернулись через час — всё на месте.

Когда дизайн готов — верхнее меню, Export. Доступны шесть форматов:

  • PDF — печать, отправка по email, документы.
  • PPTX — для презентаций, особенно если ваш руководитель/клиент любит PowerPoint.
  • Canva — импорт прямо в ваш Canva-аккаунт для дальнейшей шлифовки маркетологом.
  • HTML — одна страница в виде сайта. Можно открыть в браузере, выложить.
  • Folder — папка с ассетами (PNG, SVG, исходники), для дальнейшей работы.
  • Internal URL — ссылка прямо на ваш дизайн в Claude Design. Удобно делиться внутри организации без скачивания.

Кнопка Share рядом с Export — даёт ссылку для коллег. Три уровня доступа:

  1. Private — видите только вы.
  2. View-only — коллеги смотрят и комментируют, но не правят.
  3. Edit access — коллеги могут правки и писать в тот же диалог с Claude. Это и есть «групповой разговор», который мы разберём в этапе 09.

На этом этапе экспортируйте только в PDF — этого достаточно. Остальные форматы появятся в следующих этапах по мере необходимости.

There is no “Save” button in Claude Design. Every edit lands in the project history automatically. Close the tab, come back in an hour — everything is there.

When the design is ready — top menu, Export. Six formats:

  • PDF — print, email, documents.
  • PPTX — for slide decks, especially if your boss/client loves PowerPoint.
  • Canva — import straight into your Canva account for marketer-side polish.
  • HTML — one page as a site. Open in any browser, publish anywhere.
  • Folder — a folder of assets (PNG, SVG, sources) for further work.
  • Internal URL — a link straight to your design inside Claude Design. Useful for in-org sharing without downloads.

The Share button next to Export — gives a link for colleagues. Three access levels:

  1. Private — only you.
  2. View-only — colleagues read and comment, do not edit.
  3. Edit access — colleagues edit and write into the same dialogue with Claude. The “group conversation” we cover in etape 09.

For this etape, export to PDF only — that is enough. The other formats arrive in later etapes as needed.

Полевые правила: что Claude Design умеет, а что — пока нетField rules: what Claude Design does well, and what it does not yet

Чтобы не разочаровываться, держите в голове, что Claude Design — это research preview. Несколько правил:

  • Хорошо умеет: лендинги, обложки, постеры, питч-деки, инфографика, маркетинговые ассеты, прототипы интерфейсов, шаблонная типографика.
  • Сносно умеет: 3D-визуализация интерьера и продукта (это новинка, иногда сбивается с пропорций), анимация переходов, иконки в одном стиле.
  • Не умеет: фотореалистичный рендер уровня Corona / V-Ray, видео-монтаж, 3D-моделирование персонажей, ретушь фотографий. Это работа специальных инструментов.

Если просите что-то из «не умеет» — Claude вежливо предложит обходной путь. Например, на «сделай рендер квартиры как в каталоге IKEA» — ответит, что фотореалистичного рендера он не даёт, но может собрать стилизованный 3D-тур с переключением материалов и освещения. Часто этого достаточно для согласования с клиентом до этапа реализации.

Главное правило: если у вас уже есть Figma или Adobe и вы знаете, как ими пользоваться, для финальной шлифовки часто разумнее экспортировать туда. Claude Design — это «дизайнер первой версии и вариантов», а не замена всего pipeline. Этому посвящён этап 10 (опц.) с Canva-pipeline.

To avoid disappointment, hold this in mind: Claude Design is a research preview. A few rules:

  • Does well: landings, covers, posters, pitch decks, infographics, marketing assets, UI prototypes, template-driven typography.
  • Decent at: interior and product 3D (new, sometimes off on proportions), motion transitions, one-family icon sets.
  • Does not do: photoreal rendering at Corona/V-Ray level, video editing, 3D character modelling, photo retouching. Specialist tools own that work.

If you ask for something it does not do — Claude politely offers a workaround. Asked for “an IKEA-catalogue render of the flat” it will reply that photoreal rendering is not its job, but it can build a stylised 3D tour with switchable materials and lighting. Often enough to get a client sign-off before production work begins.

The main rule: if you already own Figma or Adobe and know how to drive them, for final polish it is often smart to export there. Claude Design is a “first-version-and-variants designer,” not a whole-pipeline replacement. Etape 10 (optional) covers the Canva pipeline for that reason.

§ II · Практика по ходуPractice along the way

Десятиминутный прогон по workspaceA ten-minute workspace warm-up

Прежде чем браться за проекты, сделайте небольшую разминку прямо сейчас. Это занимает 5–10 минут и ставит на место всё, о чём вы только что прочитали.

Before tackling the projects, take a brief warm-up right now. It takes 5–10 minutes and locks in everything you just read.

Прогон № 1 · Открываем, оглядываемся, делаем «hello»Run № 1 · Open, look around, ship a “hello”

  1. Откройте claude.ai/design. Если видите сообщение «not available» — вернитесь к теме «Доступ для Enterprise».
  2. Найдите три зоны: диалог слева, холст в центре, меню сверху. Просто покажите их себе пальцем — запомните, где что.
  3. Нажмите «New project». В диалог напишите одно предложение: «квадратная обложка плейлиста для весенних утренних прогулок, акварельный стиль».
  4. Дождитесь, пока Claude соберёт первую версию.

§ VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

Постарайтесь сначала ответить мысленно или вслух — и только потом раскрыть «правильный ответ». После того как сверились, отметьте: знал или не знал.

Try answering mentally or out loud first — only then reveal the “right answer.” After comparing, mark it: knew or did not know.

  1. Что переносится из Claude Code в Claude Design напрямую, а что меняется? What ports directly from Claude Code into Claude Design, and what changes?

    показать ответshow answer

    Переносится: модель (Opus 4.7), подписка, итерация по диалогу, конкретные референсы лучше эпитетов, приём «спросите Claude напоследок». Меняется: среда (браузер вместо CLI), медиум (холст вместо файлов), три инструмента уточнения вместо одного, native экспорт в дизайн-форматы, persistent history (без git).

    Ports over: the model (Opus 4.7), subscriptions, dialogue-driven iteration, “concrete references beat epithets,” the “ask Claude last” move. Changes: the environment (browser, not CLI), medium (canvas, not files), three refinement tools instead of one, native exports to design formats, persistent history (no git needed).

  2. По какому адресу живёт Claude Design и на какой модели работает? Where does Claude Design live and which model runs it?

    показать ответshow answer

    claude.ai/design, работает на Opus 4.7 — самой мощной vision-модели Claude на момент запуска (17 апреля 2026).

    claude.ai/design, on Opus 4.7 — the most capable Claude vision model at launch (17 April 2026).

  3. Какие подписки Anthropic дают доступ к Claude Design? Which Anthropic subscriptions give access to Claude Design?

    показать ответshow answer

    Pro, Max, Team, Enterprise. На Enterprise дополнительно нужно, чтобы администратор включил Claude Design в настройках организации — по умолчанию выключено.

    Pro, Max, Team, Enterprise. Enterprise additionally requires the org admin to enable Claude Design in organisation settings — off by default.

  4. Какие три инструмента уточнения есть в Claude Design и для чего каждый? What are the three refinement tools in Claude Design, and what is each for?

    показать ответshow answer

    Текстовый промпт — для смысловых правок («сделай контрастнее, добавь блок»). Инлайн-комментарий — для точечных задач на конкретном элементе («этот — меньше»). Слайдер — для тонкой подстройки spacing, color, layout без слов.

    Text prompt — for semantic edits (“higher contrast, add a block”). Inline comment — for pointed tasks on a specific element (“this — smaller”). Slider — for wordless fine-tuning of spacing, colour, layout.

  5. Какие три слоя чтения дизайна и каким инструментом чинится каждый? What are the three reading layers, and which tool fixes each?

    показать ответshow answer

    Композиция (что где, баланс холста) — чините инлайн-комментарием на конкретном элементе. Иерархия (что главное, в каком порядке читается) — чините текстовым промптом, это смысловая правка. Атмосфера (какой mood, тёплый/холодный, плотный/воздушный) — чините слайдером, это финальная отделка.

    Composition (what goes where, canvas balance) — fix with an inline comment on the specific element. Hierarchy (what matters, the reading order) — fix by text prompt, a semantic edit. Atmosphere (the mood, warm/cool, dense/airy) — fix with a slider, final polish.

  6. Что Claude Design не умеет делать — для чего нужны другие инструменты? What does Claude Design not do — what still needs other tools?

    показать ответshow answer

    Фотореалистичный 3D-рендер уровня Corona / V-Ray, видео-монтаж, 3D-моделирование персонажей, ретушь фотографий. Стилизованный 3D-тур и иконки в одном стиле — уже умеет, хотя ещё иногда сбивается.

    Photoreal 3D rendering at Corona/V-Ray level, video editing, 3D character modelling, photo retouching. Stylised 3D tours and one-family icon sets — it can do, though it still trips up sometimes.

  7. UI Claude Design отличается от того, что описано в тьюториале. Что делать? The Claude Design UI differs from the tutorial’s description. What now?

    показать ответshow answer

    Это нормально для research preview — интерфейс меняется. Принципы остаются: workspace, холст, три инструмента уточнения, экспорт, share, handoff. Кнопки могут переименовываться или переезжать. Если не найдёте функцию — спросите Claude в том же диалоге: «как сейчас в интерфейсе привязать комментарий к конкретному элементу?» — и он покажет.

    Normal for a research preview — the interface drifts. Principles persist: workspace, canvas, three refinement tools, export, share, handoff. Buttons may rename or relocate. If you cannot find a feature — ask Claude in the same dialogue: “how do I attach a comment to a specific element in the current interface?” It will show you.
