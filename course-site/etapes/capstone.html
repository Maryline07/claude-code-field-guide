§ I · Правила капстоунаCapstone rules

Что общее у всех четырёх вариантовWhat all four options share

Какой бы вариант вы ни выбрали, есть требования, общие для всех. Они отличают капстоун от «ещё одного учебного проекта» — и превращают результат в продукт.

Whichever option you pick, there are shared requirements. They separate the capstone from “yet another learning project” — and turn the result into a product.

  1. Spec.md обязательный. До первой строки кода — написанный спек на ~1 страницу: цель, целевой пользователь, входы, выходы, фазы, критерии готовности, что вне scope.spec.md mandatory. Before the first line of code — a one-page spec: goal, target user, inputs, outputs, phases, acceptance, out of scope.
  2. Публичный URL. Ваш продукт должен открываться по ссылке, доступной любому. Vercel/Render/GitHub Pages — на ваш выбор, в зависимости от стека.Public URL. Your product opens at a URL anyone can hit. Vercel/Render/GitHub Pages — your call by stack.
  3. Двуязычный README. ru + en, с демо-ссылкой, скриншотами или asciinema, лицензией, дисклеймером.Bilingual README. ru + en, with demo URL, screenshots or asciinema, licence, disclaimer.
  4. Минимум 6 техник курса. В продукте должны явно использоваться: spec → Plan Mode → CLAUDE.md, MCP или skill из этапа 03, frontend-design (если есть UI), верификация через Playwright, авто-расписание (cron или schedule skill), push в Telegram.At least 6 course techniques. The product must visibly use: spec → Plan Mode → CLAUDE.md, an MCP or skill from etape 03, frontend-design (if there’s UI), Playwright verification, auto-schedule (cron or the schedule skill), Telegram push.
  5. Безопасность. Никаких секретов в коммитах, .env в .gitignore, env vars через интерфейсы Vercel/Render. На каждом коммите — git diff --staged | grep -iE "key|secret|token" в hook'е.Security. No secrets in commits, .env in .gitignore, env vars via Vercel/Render UIs. On each commit — git diff --staged | grep -iE "key|secret|token" in a hook.
  6. Мониторинг. PostHog LLM Analytics включён. На дашборде вы видите токены, стоимость, ошибки в реальном времени.Monitoring. PostHog LLM Analytics on. The dashboard shows tokens, cost, errors in real time.
  7. Реальный пользователь. Покажите продукт минимум одному человеку (другу, родственнику, коллеге), послушайте 30-секундный отзыв, зафиксируйте в FEEDBACK.md.Real user. Show the product to at least one person (friend, family, colleague), capture a 30-second piece of feedback, save to FEEDBACK.md.
  8. Ретроспектива. В конце — короткий retro.md: 3 что получилось, 3 что нет, 3 что бы сделали по-другому в следующий раз.Retrospective. At the end — a short retro.md: 3 wins, 3 misses, 3 things you would do differently next time.
§ II · Четыре вариантаFour options

Выберите тот, который вам интересен личноPick the one that matters to you personally

Главное правило выбора: будете ли вы пользоваться этим сами после курса? Если да — берите. Если только «в портфолио» — будет вяло. Капстоун — единственный шанс сделать ваш собственный инструмент, к которому вы будете возвращаться годами.

The main rule of choice: will you actually use it after the course? If yes — go. If just for “the portfolio” — it’ll feel flat. The capstone is your single shot to build your own tool you’ll return to for years.

Вариант A · «Мой инвестиционный ИИ»Option A · “My Investing AI”

Полный SaaS для себя и двух-трёх друзейFull SaaS for yourself and two-three friends

Идея: расширение этапа 06 до настоящего продукта. Учёт портфеля + импорт CSV-выписок брокера + дашборд с живыми ценами + аналитика по сценариям + AI-консультант на Claude API + ежедневная Telegram-сводка + еженедельный due diligence по одному тикеру через subagents.

Idea: scale up etape 06 into a real product. Portfolio tracking + broker-CSV import + live-price dashboard + scenario analytics + Claude-API advisor + daily Telegram digest + weekly DD on one ticker via subagents.

Кто пользователь

Target user

  • вы сами — ежедневно;
  • 2–3 близких, кому интересно — через ваш сервис под их аккаунтами;
  • не широкая публика — никаких маркетинговых лендингов и подписок.
  • yourself — daily;
  • 2–3 close people who care — through your service with their accounts;
  • not the public — no marketing landings, no subscriptions.

Главные фичи

Core features

  • Регистрация по email + JWT (этап 06).
  • Импорт CSV из 3+ брокеров с универсальной нормализацией (этап 02).
  • Дашборд с живыми ценами через ваш Daily Trader MCP (этап 09).
  • Прогнозы в трёх сценариях (этап 04).
  • AI-консультант с долговременной памятью (этап 06 + этап 11).
  • Ежедневная Telegram-сводка по cron (этап 10).
  • Еженедельный DD: команда из 4 subagents (этап 07).
  • Установленный плагин @you/personal-office (этап 09) — поверх сервиса.
  • Email + JWT signup (etape 06).
  • CSV import from 3+ brokers with universal normalisation (etape 02).
  • Live-price dashboard via your Daily Trader MCP (etape 09).
  • Three-scenario forecasts (etape 04).
  • AI advisor with long-term memory (etapes 06 + 11).
  • Daily Telegram digest on cron (etape 10).
  • Weekly DD: a 4-subagent team (etape 07).
  • Your installed @you/personal-office plugin (etape 09) on top.

Стек

Stack

  • FastAPI на Render + PostgreSQL на Neon.
  • Next.js на Vercel.
  • Claude API для AI-консультанта (Sonnet) и DD (Opus для вердикта, Haiku для парсинга твитов).
  • Daily Trader MCP — для цен.
  • Системный cron или skill schedule для ежедневных и еженедельных задач.
  • PostHog для мониторинга.
  • FastAPI on Render + PostgreSQL on Neon.
  • Next.js on Vercel.
  • Claude API for the advisor (Sonnet) and DD (Opus for verdict, Haiku for tweet parsing).
  • Daily Trader MCP for prices.
  • System cron or the schedule skill for daily and weekly tasks.
  • PostHog for monitoring.

Критерии готовности

Acceptance criteria

  • сервис открывается по публичному URL и вы даёте доступ другу;
  • друг регистрируется, импортирует свою выписку, видит свой дашборд;
  • его дашборд отделён от вашего — полная изоляция данных;
  • в течение 7 дней Telegram-сводка приходит каждый день;
  • пятничный DD пришёл хотя бы один раз;
  • вы использовали продукт сами 5+ дней подряд — не «построил и забыл», а «стал частью утра»;
  • есть FEEDBACK.md с 30-секундным отзывом друга и вашими ответами на него.
  • the service opens at a public URL, you grant access to a friend;
  • the friend signs up, imports their statement, sees their own dashboard;
  • their dashboard is isolated from yours — full data segregation;
  • over 7 days the Telegram digest lands daily;
  • the Friday DD landed at least once;
  • you used the product yourself for 5+ consecutive days — not “build and forget,” but “part of the morning”;
  • FEEDBACK.md captures a 30-second piece of feedback from a friend and your responses.
Вариант B · «Налоговая студия»Option B · “Tax Studio”

08b → многопользовательский веб-сервис08b → a multi-user web service

Идея: масштабирование проекта 08b. Был «один пайплайн на вашу декларацию» — станет веб-сервис, в который любой может загрузить ZIP с первичкой, очередь её обработает, и через 5–10 минут вернёт filled-1040.pdf и memo.md.

Idea: scale up project 08b. Was “one pipeline for your return” — becomes a web service where anyone uploads a ZIP of primary docs, the queue processes it, and 5–10 minutes later returns filled-1040.pdf and memo.md.

Кто пользователь

Target user

  • люди, кто делает 1040 сами (US filers со стандартными доходами);
  • CPA, желающие ускорить «черновик» перед ревью;
  • не замена налогового бухгалтера — помощник для первого черновика.
  • people doing their own 1040 (US filers with vanilla incomes);
  • CPAs wanting a faster “draft” before review;
  • not a replacement for a CPA — a first-draft assistant.

Главные фичи

Core features

  • Регистрация + JWT (этап 06).
  • Загрузка ZIP с первичкой; обработка в фоновой задаче внутри web-сервиса (приём из этапа 06b) либо через очередь Celery/RQ.
  • Парсинг через параллельные subagents (этап 07).
  • Расчёт через ваш скилл «налоговый аналитик» — адаптация скилла из этапа 08a.
  • Заполнение fillable-1040 через skill pdf (этап 08b).
  • Аккаунты для CPA: видят список «ваших клиентов» с их черновиками.
  • Аудит-лог с source → field → value для каждой строки 1040.
  • Дисклеймер на каждой странице.
  • Signup + JWT (etape 06).
  • ZIP upload of primary docs; processing as a background task inside the web service (the etape-06b trick) or via a Celery/RQ queue.
  • Parsing via parallel subagents (etape 07).
  • Calculation through your “tax analyst” skill — adapted from etape 08a.
  • Filling fillable-1040 via the pdf skill (etape 08b).
  • CPA accounts: a list of “your clients” with drafts.
  • Audit log with source → field → value for every 1040 line.
  • Disclaimer on every page.

Критерии готовности

Acceptance criteria

  • 5 разных тестовых ZIP-ов проходят полный пайплайн без вашего вмешательства;
  • результат для каждого — PDF + memo + audit.log за 5–10 минут;
  • в логах нет персональных данных пользователей (анонимизация);
  • файлы пользователя удаляются автоматически через 24 часа;
  • тестовый CPA-аккаунт видит свои тестовые «клиенты»;
  • есть очевидный дисклеймер «не для подачи в IRS без CPA».
  • 5 different test ZIPs pass the full pipeline without your intervention;
  • each result — PDF + memo + audit.log within 5–10 minutes;
  • logs carry no personal data (anonymisation);
  • user files auto-delete after 24 hours;
  • a test CPA account sees its test “clients”;
  • an obvious disclaimer “not for IRS filing without a CPA.”
Вариант C · «Семейный финансовый портал»Option C · “Family Finance Portal”

Бюджет, цели и расходы для всей семьиBudget, goals and expenses for the whole family

Идея: общий финансовый пульт для вашей семьи (вы + партнёр, или вы + родители). Совместный бюджет, цели накоплений, расписание оплат, Telegram-бот для быстрого ввода трат, расходы по категориям с автоматической классификацией через Claude.

Idea: a shared financial cockpit for your family (you + partner, or you + parents). Shared budget, saving goals, payment schedule, a Telegram bot for fast expense entry, categorised expenses with Claude-driven auto-classification.

Кто пользователь

Target user

  • вы и ещё 1–3 члена семьи;
  • не для широкого использования — данные слишком чувствительные;
  • цель — «семейная финансовая грамотность» через инструмент.
  • you and 1–3 family members;
  • not for broad use — data is too sensitive;
  • goal — “family financial literacy” via a tool.

Главные фичи

Core features

  • Multi-user аккаунты с ролями (admin / member).
  • Telegram-бот: пишете «обед 800» — бот через Claude API классифицирует «еда вне дома» и записывает.
  • Совместный дашборд: где общие траты, где — индивидуальные.
  • Цели накоплений: «отпуск 200к к июлю», прогресс-бар, прогноз достижения (этап 04).
  • Расписание счетов (электричество, ипотека) с cron-напоминалкой за 3 дня (этап 10).
  • Еженедельный Sunday-отчёт по всей семье через subagents (этап 07).
  • Шифрование чувствительных полей (этап 10).
  • Multi-user accounts with roles (admin / member).
  • Telegram bot: type “lunch 800” — Claude API classifies “eating out” and saves.
  • Shared dashboard: shared vs personal expenses.
  • Saving goals: “vacation 200k by July,” progress bar, ETA forecast (etape 04).
  • Bill schedule (utilities, mortgage) with a 3-day cron reminder (etape 10).
  • Sunday weekly family report via subagents (etape 07).
  • Encryption of sensitive fields (etape 10).

Критерии готовности

Acceptance criteria

  • в портал залогинились минимум 2 человека (вы и кто-то ещё);
  • оба пользуются Telegram-ботом для ввода трат;
  • Claude корректно классифицирует 9 из 10 трат;
  • обе цели накоплений видны на дашборде, прогресс обновляется автоматически;
  • Sunday-отчёт пришёл за неделю наблюдений;
  • персональные финансовые данные зашифрованы в БД, есть /forget.
  • at least 2 people signed in (you + someone);
  • both use the Telegram bot for expenses;
  • Claude correctly classifies 9 out of 10 entries;
  • both saving goals show on the dashboard, progress auto-updates;
  • Sunday report landed for a week of observations;
  • sensitive financial data encrypted in the DB, with a /forget command.
Вариант D · «Совместный AI-фонд»Option D · “Co-op AI Fund”

DAO для инвестиций без блокчейнаA DAO for investing — without blockchain

Идея: групповой инвестиционный сервис для кружка из 5–15 человек (друзей, коллег). Каждый предлагает идею через Telegram-бота, AI-команда из этапа 07 делает due diligence, голосование, агрегированные позиции, ежемесячный отчёт. Никаких реальных денег — это игровой фонд для тренировки коллективного инвест-мышления.

Idea: a group investing service for a circle of 5–15 (friends, colleagues). Each member proposes via a Telegram bot, the etape-07 AI team runs DD, voting, aggregated positions, a monthly report. No real money — a game fund to practise collective investment thinking.

Кто пользователь

Target user

  • кружок из 5–15 человек, который хочет «играть в фонд»;
  • не реальный фонд — учебный полигон;
  • можно использовать как «инвестиционный книжный клуб» — обсуждение через AI-аналитику.
  • a circle of 5–15 wanting to “play fund”;
  • not a real fund — a learning playground;
  • doubles as an “investing book club” — debate via AI analytics.

Главные фичи

Core features

  • Multi-user сервис: frontend на Vercel, backend на Render, БД на Neon.
  • Telegram-бот: /propose AAPL because ... — идея уходит в очередь.
  • AI-команда (4 subagents этапа 07) автоматически делает DD по каждой идее, отчёт виден всем.
  • Голосование: каждый член ставит +1/0/-1 на идею; агрегация решает «принимаем / нет».
  • Виртуальный портфель: фиксированные «фонд-баксы», распределяются по принятым идеям.
  • Ежемесячный отчёт: какие идеи сработали, какие — нет, кто из участников «лучший аналитик» по hit-rate.
  • Полный «red-team» в конце каждого DD (этап 07).
  • A multi-user service: frontend on Vercel, backend on Render, DB on Neon.
  • Telegram bot: /propose AAPL because ... — the idea enters the queue.
  • The AI team (4 etape-07 subagents) auto-DDs each idea, the report visible to all.
  • Voting: each member casts +1/0/-1; aggregation decides accept/reject.
  • Virtual portfolio: fixed “fund-bucks” allocated to accepted ideas.
  • Monthly report: which ideas worked, which didn’t, who is the “best analyst” by hit-rate.
  • Full “red team” at the end of each DD (etape 07).

Критерии готовности

Acceptance criteria

  • в сервисе зарегистрировано минимум 5 участников;
  • хотя бы 3 идеи прошли полный цикл «proposal → DD → vote → portfolio»;
  • пришёл первый ежемесячный отчёт;
  • «лучший аналитик» определён корректно (по hit-rate, а не по количеству);
  • есть очевидный дисклеймер «игровой фонд, никаких реальных денег»;
  • Friend feedback: 2 участника пишут одно предложение «почему мне нравится / не нравится» в FEEDBACK.md.
  • at least 5 members signed up;
  • at least 3 ideas went through the full cycle “proposal → DD → vote → portfolio”;
  • the first monthly report has landed;
  • “best analyst” is determined correctly (by hit-rate, not count);
  • obvious disclaimer: “game fund, no real money”;
  • Friend feedback: 2 members write a one-liner “what I like / don’t” into FEEDBACK.md.

Сквозные навыки

Cross-cutting skills

  • Промпт-инжиниринг для агентов: формулировка → итерация → откат → варианты → аудит.
  • Дисциплина крупных задач: spec → Plan Mode → TodoWrite → verification → /compact.

  • Расширение инструмента: свои skills, slash, hooks, MCP, плагины.
  • Полный цикл продукта: frontend + backend + БД + auth + деплой + мониторинг + автопилот.
  • Prompt engineering for agents: phrasing → iteration → rollback → variants → audit.
  • Big-task discipline: spec → Plan Mode → TodoWrite → verification → /compact.
  • Agent teams: parallel subagents, briefs, conductor, red team.
