§ I · ТемыTopics

Что у Claude теперь в рукахWhat Claude now has in hand

Что такое MCP. Почему через него — всёWhat MCP is, and why everything goes through it

MCP (Model Context Protocol) — это стандарт «инструментов для AI». Договорились: если кто-то делает сервис, который AI должен уметь использовать, он оформляет его как «MCP-сервер». Любой AI-клиент (включая Claude Code), поддерживающий MCP, мгновенно умеет с этим сервисом работать.

Аналогия: USB. Раньше для каждой мышки нужен был свой драйвер. Сейчас мышка любой марки втыкается в любой компьютер — благодаря стандарту USB. MCP — это USB для AI-инструментов.

Что появляется у Claude после установки MCP-сервера:

  • новые «инструменты» (tools), которые он может использовать — от «открой страницу» до «запиши в PostHog»;
  • новые «ресурсы» (resources) — ссылки на документы, дашборды, базы знаний;
  • иногда — готовые промпты (prompts) для типовых задач сервиса.

В этом этапе мы ставим два MCP: playwright (браузер) и context7 (свежие доки). Этап 09 будет про то, как написать собственный MCP — например, MCP «инвест-помощник» с инструментами get_price, compute_signal, set_alert.

Реестр MCP-серверов и skills — smithery.ai. Это «магазин AI-инструментов»: тысячи серверов от разных авторов, ставятся одной командой.

MCP (Model Context Protocol) is a standard for “AI tools.” The agreement: if you build a service AI should use, you wrap it as an “MCP server.” Any MCP-aware client (including Claude Code) instantly knows how to use it.

An analogy: USB. Each mouse used to need its own driver. Now any mouse plugs into any computer — thanks to USB. MCP is the USB of AI tools.

After installing an MCP server, Claude gains:

  • new tools — from “open this page” to “write to PostHog”;
  • new resources — links to documents, dashboards, knowledge bases;
  • sometimes — pre-built prompts for the service’s typical tasks.

This etape installs two MCPs: playwright (browser) and context7 (fresh docs). Etape 09 covers building your own MCP — e.g., an “investing helper” MCP with get_price, compute_signal, set_alert.

The MCP and skills registry is smithery.ai. An “AI tools store”: thousands of servers from many authors, one-command install.
WebSearch и WebFetch — встроенные «руки в интернет»WebSearch and WebFetch — built-in “hands into the internet”

Эти два инструмента у Claude Code уже есть — ставить ничего не нужно. Разница простая:

  • WebSearch: «погугли это». Запрос → список ссылок и коротких сниппетов.
  • WebFetch: «прочитай эту конкретную страницу». URL → текст с неё (HTML очищен).

Типичный сценарий:

> найди (WebSearch) последний квартальный отчёт Apple,
> потом открой (WebFetch) первую ссылку из investor.apple.com
> и выпиши три ключевых цифры в data/aapl-q.md.

Когда уместен WebSearch:

  • не знаете точный URL;
  • нужно несколько источников;
  • задача исследовательская.

Когда WebFetch:

  • знаете URL и нужен именно он;
  • задача — вытащить данные из конкретной страницы.
WebFetch не подходит для интерактивных страниц (с кучей JavaScript, login и т. д.). Для них — Playwright (см. ниже).

These two are already in Claude Code — nothing to install. The split:

  • WebSearch: “google this.” Query → list of links and short snippets.
  • WebFetch: “read this exact page.” URL → cleaned text.

Typical pattern:

> find (WebSearch) Apple's latest quarterly report,
> then open (WebFetch) the first investor.apple.com link
> and pull three key numbers into data/aapl-q.md.

WebSearch fits when:

  • you don’t know the URL;
  • you need several sources;
  • the task is exploratory.

WebFetch fits when:

  • you know the URL and want exactly it;
  • the task is to extract data from a specific page.
WebFetch is not great for interactive pages (heavy JS, login). For those — Playwright (below).
Playwright: Claude управляет настоящим браузеромPlaywright: Claude controlling a real browser

Playwright — это когда Claude открывает настоящий Chrome (через MCP), кликает кнопки, заполняет формы, делает скриншоты. То, чего не умеют WebSearch и WebFetch.

Главные применения в курсе:

  1. Проверка своего сайта. Сделали дашборд — Claude открывает его, делает скриншот, говорит «вот тут график обрезан, вот тут заголовок переносится плохо».
  2. Парсинг сайтов без API. Например, «сходи на страницу акции на Yahoo Finance, выпиши P/E ratio». Если открытого API нет — парсим браузером.
  3. Тестирование. «Зарегистрируйся на своём же сайте через тестовый email, проверь что приходит письмо, нажми ссылку из него, проверь что вошёл».
> через playwright открой http://localhost:8000/dashboard.html
> в окне 1440x900, сделай скриншот, оцени:
> — нет ли элементов, налезающих друг на друга;
> — читаются ли числа в KPI-плитках;
> — все ли графики прорисовались.

Playwright is Claude opening a real Chrome (via MCP), clicking buttons, filling forms, taking screenshots. Things WebSearch / WebFetch cannot do.

Main uses in this course:

  1. Verifying your own site. Built a dashboard — Claude opens it, screenshots, says “the chart is clipped here, the title wraps badly there.”
  2. Parsing sites without an API. E.g., “go to a stock’s page on Yahoo Finance, pull the P/E.” If no open API — scrape via the browser.
  3. Testing. “Sign up on your own site with a test email, check the email arrives, click the link, confirm you’re in.”
> through playwright open http://localhost:8000/dashboard.html
> at 1440x900, screenshot, judge:
> — anything overlapping?
> — numbers in KPI tiles readable?
> — all charts rendered?
Context7: чтобы Claude не писал код по старым версиямContext7: keeping Claude off stale library versions

Внутри Claude — знания «до даты обучения» (cutoff). За это время многие библиотеки выпустили новые версии, поменяли API. Если Claude помнит «как было», его код может не запуститься.

Context7 решает это так: даёт Claude доступ к актуальной документации тысяч библиотек в момент запроса. Claude сам читает свежий API и пишет под него.

Когда вызывать вручную:

  • задача завязана на конкретную библиотеку (chart.js, react, fastapi, prisma и т. д.);
  • вы видите, что Claude использует устаревший синтаксис;
  • вы на «фронтире» технологии — что-то новое.
> через context7 подтяни доку chart.js v4
> и сделай тёмную тему графика,
> используя именно текущий API.

Когда не нужно: для общих задач (HTML, CSS, базовый JS, общая логика). Там Claude и так точен.

Claude knows things up to its training cutoff. Since then libraries shipped new versions, changed APIs. If Claude remembers “how it was,” the code may not run.

Context7 fixes this: it gives Claude live access to current docs of thousands of libraries at request time. Claude reads the fresh API and writes against it.

Invoke explicitly when:

  • the task hinges on a specific library (chart.js, react, fastapi, prisma…);
  • you see Claude using stale syntax;
  • you’re on the frontier — something new.
> through context7 pull chart.js v4 docs
> and make a dark-theme chart
> using the current API.

Skip it for general tasks (HTML, CSS, basic JS, general logic) — Claude is precise there.

Открытые API: где брать живые финансовые данныеOpen APIs: where to get live financial data

API — это «розетка», в которую программа втыкается, чтобы получить данные. У большинства финансовых сервисов есть открытые API. Несколько, которые мы используем в курсе:

  • Alpha Vantage (alphavantage.co) — акции, валюты, крипта. Бесплатно: 25 запросов/день. Нужен API-ключ.
  • Yahoo Finance (через unofficial API или yfinance) — акции и ETF, без ключа.
  • CoinGecko (coingecko.com) — цены крипты, без ключа.
  • Frankfurter (frankfurter.app) — курсы валют от ЕЦБ, без ключа.

Что важно знать про API:

  • Rate limit — ограничение «сколько запросов в минуту/день». Бесплатные тарифы обычно строгие.
  • API-ключ — секретный пропуск. Никогда не коммитить в git, не писать в HTML, не показывать в скриншотах. Хранить в .env.
  • Эндпоинт — конкретный URL внутри API: например, /query?function=GLOBAL_QUOTE&symbol=AAPL.
  • Формат ответа — обычно JSON.

An API is a “socket” programs plug into to get data. Most financial services expose open APIs. A few we use in the course:

  • Alpha Vantage (alphavantage.co) — stocks, FX, crypto. Free: 25 req/day. Key required.
  • Yahoo Finance (via unofficial APIs / yfinance) — stocks and ETFs, no key.
  • CoinGecko (coingecko.com) — crypto prices, no key.
  • Frankfurter (frankfurter.app) — ECB FX rates, no key.

What you should know:

  • Rate limit — “requests per minute/day.” Free tiers are typically strict.
  • API key — a secret pass. Never commit to git, write into HTML, or show in screenshots. Keep in .env.
  • Endpoint — a concrete URL inside the API: e.g. /query?function=GLOBAL_QUOTE&symbol=AAPL.
  • Response format — usually JSON.

§ V · Словарь этапаEtape glossary

Что добавилось к словарюNew entries in the glossary

MCP
Model Context Protocol — стандарт «инструментов для AI». Через MCP-сервер Claude получает новые навыки: открывать браузер, читать свежие доки, общаться с внешним сервисом. Аналогия — USB для AI. Model Context Protocol — a standard for “AI tools.” Through MCP servers Claude gets new abilities: open the browser, read fresh docs, talk to an external service. Analogy — USB for AI.
Smithery
Реестр MCP-серверов и skills. Тысячи готовых пакетов. Skill ставится одной командой npx smithery@latest skill add <owner/skill> --agent claude-code; для MCP-серверов — smithery mcp add <url> --client claude (но для official-серверов вроде @playwright/mcp чище нативная claude mcp add). A registry of MCP servers and skills. Thousands of ready packages. A skill installs via npx smithery@latest skill add <owner/skill> --agent claude-code; an MCP server via smithery mcp add <url> --client claude (but for official servers like @playwright/mcp, native claude mcp add is cleaner).
WebSearch
Встроенный инструмент Claude Code: «погугли это». Возвращает список ссылок и коротких сниппетов. Не требует установки. A Claude Code built-in: “google this.” Returns links and snippets. No install needed.
WebFetch
Встроенный инструмент: «прочитай эту страницу». На вход URL, на выход — очищенный текст. Не подходит для интерактивных страниц — для них Playwright. A built-in: “read this page.” URL in, cleaned text out. Not great for interactive pages — Playwright handles those.
Playwright
Движок автоматизации браузера от Microsoft. Через MCP даёт Claude инструменты: открыть страницу, кликнуть, ввести текст, сделать скриншот. Используется для ревизии своих сайтов и парсинга страниц без API. Browser-automation engine by Microsoft. Through MCP it gives Claude tools: open, click, type, screenshot. Used to audit your own sites and scrape pages without an API.
Context7
MCP-сервер от Upstash, дающий Claude доступ к актуальной документации тысяч библиотек. Решает проблему «обучен на старых версиях». An Upstash MCP server giving Claude access to current docs for thousands of libraries. Solves “trained on old versions.”
API
Application Programming Interface — «программный интерфейс»: способ одной программы получить данные у другой по строгим правилам. Финансовые сервисы (Yahoo, Alpha Vantage, CoinGecko) предоставляют открытые API для цен. Application Programming Interface — how one program asks another for data by strict rules. Financial services (Yahoo, Alpha Vantage, CoinGecko) expose open APIs for prices.
API-ключAPI key
Секретная строка, которая идентифицирует вас перед API. Никогда не коммитьте в git, не пишите в HTML, не показывайте в скриншотах. Хранится в .env, который добавляется в .gitignore. A secret string identifying you to the API. Never commit, never embed in HTML, never put in screenshots. Lives in .env, which is in .gitignore.
RSS
Really Simple Syndication — стандарт «лент новостей» в формате XML. У большинства новостных сайтов есть открытые RSS-фиды — самый простой способ собирать заголовки автоматически. Really Simple Syndication — the XML standard for news feeds. Most news sites publish open RSS feeds — the simplest way to harvest headlines automatically.
Rate limit
Ограничение, сколько запросов в единицу времени можно сделать к API. Бесплатные тарифы обычно строгие. Решения: кэширование, паузы между запросами, графический fallback. A cap on “requests per minute/day” to an API. Free tiers are typically strict. Mitigations: caching, sleeps between requests, graceful fallback.
ЭндпоинтEndpoint
Конкретный URL внутри API, отвечающий за одну функцию. Например, у Alpha Vantage есть эндпоинт /query?function=GLOBAL_QUOTE&symbol=AAPL для текущей цены акции. A specific URL inside an API, owning one function. E.g., Alpha Vantage has /query?function=GLOBAL_QUOTE&symbol=AAPL for current stock price.
JSON
JavaScript Object Notation — формат обмена данными между программами. Похож на «вложенные словари». Большинство API возвращают ответ именно в JSON. JavaScript Object Notation — a data-exchange format between programs. Looks like “nested dictionaries.” Most APIs reply in JSON.
.env
Текстовый файл с секретными переменными окружения (API-ключи, пароли). Лежит в корне проекта, обязательно в .gitignore. Программы читают его через библиотеки типа python-dotenv. A text file with secret environment variables (API keys, passwords). Lives at project root, always in .gitignore. Programs read it through libraries like python-dotenv.
§ VI · Quiz

Семь вопросов на закреплениеSeven questions for reinforcement

  1. В чём разница между skill и MCP?What is the difference between a skill and an MCP?

    показать ответshow answer

    Skill — набор инструкций и шаблонов, который Claude читает и применяет. Меняет, как он думает и пишет. Ничего не запускает на вашем компьютере. MCP — работающий сервер с инструментами (открыть браузер, прочитать БД). Даёт Claude новые навыки.

    Skill — instructions and templates Claude reads and applies. Changes how it thinks and writes. Runs nothing on your machine. MCP — a running server with tools (open browser, read DB). Gives Claude new abilities.

  2. Когда лучше WebFetch, а когда Playwright?When is WebFetch enough, when do you need Playwright?

    показать ответshow answer

    WebFetch — для статичных страниц с текстом (Wikipedia, документация, блоги). Быстрее и легче. Playwright — для интерактивных (с тяжёлым JavaScript, авторизацией, динамической подгрузкой). Также для ревизии своих собственных сайтов — со скриншотами и кликами.

    WebFetch — for static text pages (Wikipedia, docs, blogs). Faster, lighter. Playwright — for interactive pages (heavy JS, login, dynamic load). Also for auditing your own sites — screenshots and clicks.

  3. Зачем нужен Context7, если Claude и так знает большинство библиотек?Why Context7, if Claude knows most libraries already?

    показать ответshow answer

    Знания Claude обрываются на дате обучения. За это время многие библиотеки выпускают новые версии и меняют API. Context7 даёт Claude актуальную документацию в момент запроса — и он пишет под текущий API, а не под тот, что помнит.

    Claude’s knowledge cuts off at training. Libraries ship new versions and break APIs since. Context7 gives Claude current docs at request time — so it codes against the current API, not what it remembers.

  4. Где хранить API-ключ Alpha Vantage и что нельзя делать с этим файлом?Where do you keep an Alpha Vantage API key, and what must you never do with that file?

    показать ответshow answer

    В файле .env в корне проекта (например, ALPHAVANTAGE_KEY=xxx). Никогда не коммитить в git, не вшивать в HTML/JS, не показывать в скриншотах. Файл должен быть в .gitignore.

    In a .env file at the project root (e.g. ALPHAVANTAGE_KEY=xxx). Never commit to git, embed in HTML/JS, or show in screenshots. The file must be in .gitignore.

  5. Что такое rate limit и три способа в него не упереться?What is a rate limit and three ways not to hit it?

    показать ответshow answer

    Rate limit — ограничение, сколько запросов в минуту/день можно сделать к API. Способы: кэширование (запоминать ответы на N минут), паузы между запросами (sleep), graceful fallback — если ответа нет, показать что-то полезное (например, последнюю кэшированную цену с пометкой «устаревшая»).

    A rate limit caps requests per minute/day to an API. Ways: caching (remember replies for N minutes), sleeps between requests, graceful fallback — if no reply, show something useful (e.g. last cached price marked “stale”).

    /mcp inside Claude Code (slash command). Likewise /skills for skills.

  7. Зачем использовать Playwright для ревизии своего же сайта?Why use Playwright to audit your own site?

    показать ответshow answer

    Чтобы Claude посмотрел на сайт глазами пользователя — не по коду, а по скриншоту. Видит ли он все элементы, читаются ли числа, не налезают ли блоки друг на друга. Это вторая, независимая «пара глаз» — катастрофически полезно, потому что код может быть «правильным», но UX — плохим.

    So Claude sees the site as a user — not from code but from a screenshot. Are all elements visible, numbers readable, blocks overlapping? It’s a second, independent pair of eyes — invaluable because the code can be “correct” while UX is bad.
