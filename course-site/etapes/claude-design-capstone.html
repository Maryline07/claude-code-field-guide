§ КапстоунCapstone  ·  Полный визуальный пакет одного продуктаFull visual package for one product

Один продукт.
Один пакет.
Публичный URL.One product.
One package.
One public URL.

Это финал. К этому моменту вы прошли 10 этапов и знаете весь арсенал Claude Design: бриф, brand system, прототип, handoff, дек, маркетинг, code-powered, командная работа. Капстоун — собрать всё это вокруг одного выбранного продукта и довести до публичной ссылки. На выходе — не абстрактный артефакт, а настоящий полный пакет, которым можно показывать клиенту, инвестору, партнёру.

The finale. By now you’ve gone through 10 etapes and own the full Claude Design arsenal: brief, brand system, prototype, handoff, deck, marketing, code-powered, teamwork. The capstone — assemble all of it around one chosen product and ship it to a public link. The output is not an abstract artefact but a real, full package you can show a client, investor, partner.

§ I · Капстоун-брифCapstone brief

Что вы строитеWhat you’re building

Капстоун — это один продукт по одному из четырёх треков. Не «перебежать по четырём за неделю» — глубоко погрузиться в один и собрать его до уровня «можно показать клиенту/инвестору».

The capstone is one product on one of four tracks. Not “sprint across four in a week” — dive deep into one and ship it to “showable to client/investor” level.

Что должно быть на выходе:

  1. Brand system — tokens.json в git, применённый ко всем артефактам.
  2. Лендинг — 1280×800 desktop + mobile 375, с OG-image, развернутый по публичному URL.
  3. Интерактивный прототип — 5+ экранов с состояниями, тестированный на 3+ живых людях.
  4. Handoff bundle → Claude Code → реальный сайт — код в git, deploy на Vercel/GitHub Pages.
  5. Питч-дек — 10–12 слайдов, PDF + PPTX, готов к инвестору/партнёру.
  6. Launch-серия — 5 постов канона (teaser/reveal/BTS/launch/proof) для одной платформы (X / IG / LinkedIn на ваш выбор).
  7. Один code-powered «вау»-элемент — shader-фон, 3D-тур, voice UI или motion-переход (на выбор по треку).
  8. Документация решений — markdown-файл, кратко описывающий: задача, целевая аудитория, ключевые дизайн-решения, что вынесли в Canva (если применимо).

What ships:

  1. Brand system — tokens.json in git, applied across all artefacts.
  2. Landing — 1280×800 desktop + mobile 375, with OG image, deployed at a public URL.
  3. Interactive prototype — 5+ screens with states, tested on 3+ real people.
  4. Handoff bundle → Claude Code → live site — code in git, deployed to Vercel/GitHub Pages.
  5. Pitch deck — 10–12 slides, PDF + PPTX, ready for investor/partner.
  6. Launch series — 5 canonical posts (teaser/reveal/BTS/launch/proof) for one platform (X / IG / LinkedIn — your call).
  7. One code-powered “wow” element — shader bg, 3D tour, voice UI, or motion transition (track-dependent).
  8. Decision documentation — markdown file briefly outlining: task, audience, key design decisions, what went to Canva (if applicable).
§ II · Четыре трека на выборFour tracks to choose from

Выберите одинPick one

Трек A · кофейняTrack A · café

Открытие крафтовой кофейни «Северный край»Opening a craft café “Northern Coast”

Полный визуальный пакет для запуска кофейни. Бренд — логотип, палитра, типографика, оформление чашек/стаканов. Лендинг — адрес, меню, бронирование столика, история кофейни. Прототип — мобильное приложение бронирования (этап 04). Дек — для франшиз-партнёра или инвестора. Соцсети — серия открытия в Instagram. Wow-элемент — стилизованная 3D-сцена обжарки в hero лендинга.

Full visual pack for a café launch. Brand — logo, palette, typography, cup/bag design. Landing — address, menu, table booking, café story. Prototype — mobile booking app (etape 4). Deck — for franchise partner or investor. Social — opening series for Instagram. Wow — a stylised 3D coffee-roasting scene in the landing hero.

Трек B · квартираTrack B · apartment

Дизайн-проект квартиры под ключTurn-key apartment design project

Полный пакет для презентации клиенту. Бренд — идентичность вашего дизайн-бюро (личный бренд или вымышленное). Лендинг — портфолио бюро. Прототип — конфигуратор отделки квартиры (этап 04). Дек — для тендера или презентации проекта клиенту. Соцсети — reveal-серия одного готового проекта (до/после). Wow-элемент — интерактивный 3D-тур по концепту с переключением материалов и освещения.

Full pack for client pitching. Brand — identity of your bureau (personal or imaginary). Landing — the bureau’s portfolio. Prototype — finishes configurator (etape 4). Deck — for a tender or client project pitch. Social — reveal series for one finished project (before/after). Wow — interactive 3D tour of a concept with material/lighting switches.

Трек C · играTrack C · game

Запуск инди-игры в SteamIndie game Steam launch

Полный пакет для Steam-релиза. Бренд — студия (Aurora Pixel из этапа 03 или своя). Лендинг — tideofstars.com или ваш реальный домен, с Steam-CTA, фичами, скриншотами. Прототип — UI меню игры (этап 04). Дек — для инвестора или издателя. Соцсети — launch-серия для X (Twitter) и Steam-страницы. Wow-элемент — shader-фон в hero лендинга (звёздное небо с переливами).

Full pack for a Steam release. Brand — studio (Aurora Pixel from etape 3 or yours). Landing — tideofstars.com or your real domain, with Steam CTA, features, screenshots. Prototype — game menu UI (etape 4). Deck — for investor or publisher. Social — launch series for X and the Steam page. Wow — shader background in the landing hero (starlit sky with shifts).

Трек D · финансыTrack D · finance

Личный инвест-кабинет (мост к основному курсу)Personal Investment Office (bridge to the main course)

Финальный визуальный пакет для финансового продукта, продолжающего этап 06 основного курса. Бренд — FT-style editorial. Лендинг — для пользователей-инвесторов. Прототип — кабинет с дашбордом, прогнозом, экспортом (этап 04). Дек — для инвест-фонда или партнёра. Соцсети — серия для Telegram-канала с еженедельными апдейтами портфеля. Wow-элемент — motion-переходы между сценариями прогноза (pessimistic / base / optimistic).

Final visual pack for a financial product extending main-course etape 6. Brand — FT-style editorial. Landing — for investor-users. Prototype — the office with dashboard, forecast, export (etape 4). Deck — for an investment fund or partner. Social — series for a Telegram channel with weekly portfolio updates. Wow — motion transitions between forecast scenarios (pessimistic / base / optimistic).

§ III · Этапы капстоунаCapstone milestones

Семь дней — семь milestoneSeven days, seven milestones

Если хотите минимальный темп — семь рабочих дней. Если хотите глубоко — растягивайте до двух недель.

  1. День 1 · Бриф и бренд. Напишите .docx-бриф (раздел контекста, цели, аудитория, тон, структура — этап 02). Создайте workspace, постройте brand-system через диалог или из готового кода — 5–10 tokens (этап 03). Экспорт: tokens.json + brand.css в git.
  2. День 2 · Лендинг. Главная страница продукта (этапы 01–02): bif прикреплён, tokens из workspace применены, структура из четырёх слоёв. Mobile-версия обязательна. OG-image (этап 07). Экспорт: HTML.
  3. День 3 · Прототип. 5-экранный кликабельный прототип с графом навигации, состояниями (default/empty/error), на workspace tokens (этап 04). Тест на 3 живых людях по think-aloud. Записать инсайты в diary.md.
  4. День 4 · Handoff & live URL. Handoff bundle экспортирован, передан в Claude Code (этап 05). Реализация в коде, деплой на Vercel/GitHub Pages. У вас публичный URL.
  5. День 5 · Питч-дек. 10–12 слайдов из .xlsx с метриками (этап 06). Канон cover/problem/solution/market/traction/team/ask. Экспорт PDF + PPTX.
  6. День 6 · Launch-серия. 5 постов канона teaser/reveal/BTS/launch/proof для одной платформы (этап 07). В workspace tokens. Экспорт PNG.
  7. День 7 · Wow-элемент & документация. Один code-powered элемент по вашему треку (этап 08). Markdown-документ с ключевыми решениями (этап 09 sum-комментарий-лист). Финальный коммит. Отправить ссылку на живой URL минимум одному реальному человеку.
  1. Day 1 · Brief & brand. Write a .docx brief (context, goals, audience, tone, structure — etape 2). Make a workspace, build a brand system via dialogue or from existing code — 5–10 tokens (etape 3). Export: tokens.json + brand.css to git.
  2. Day 2 · Landing. Main product page (etapes 1–2): brief attached, workspace tokens applied, four-layer structure. Mobile required. OG image (etape 7). Export: HTML.
  3. Day 3 · Prototype. 5-screen clickable prototype with navigation graph, states (default/empty/error), on workspace tokens (etape 4). Test on 3 real people via think-aloud. Write insights into diary.md.
  4. Day 4 · Handoff & live URL. Handoff bundle exported, sent to Claude Code (etape 5). Implementation in code, deploy to Vercel/GitHub Pages. You hold a public URL.
  5. Day 5 · Pitch deck. 10–12 slides from .xlsx metrics (etape 6). Canon cover/problem/solution/market/traction/team/ask. Export PDF + PPTX.
  6. Day 6 · Launch series. 5 canon posts teaser/reveal/BTS/launch/proof for one platform (etape 7). In workspace tokens. Export PNG.
  7. Day 7 · Wow element & docs. One code-powered element per your track (etape 8). Markdown document with key decisions (etape 9 summary sheet). Final commit. Send the live URL to at least one real person.
§ IV · Критерии «готово»“Done” criteria

Когда капстоун закрытWhen the capstone closes

Капстоун закрыт, когда выполнены семь объективных условий:

  • Все 7 артефактов выше существуют как файлы (или live URL).
  • Бренд системно применён ко всем 7 артефактам (видна «семья»).
  • Лендинг и прототип развернуты по публичным URL.
  • Прототип протестирован на 3+ живых людях, инсайты в diary.md.
  • Handoff bundle прошёл через Claude Code и превратился в живой код в git.
  • Дек прочитан хотя бы одним другим человеком, и он сказал «понял, в чём суть, за 3 минуты».
  • Wow-элемент работает в браузере на mobile (на вашем телефоне).

The capstone closes when seven objective conditions are met:

  • All 7 artefacts above exist as files (or live URLs).
  • The brand is systematically applied across all 7 (visible “family”).
  • Landing and prototype deployed at public URLs.
  • Prototype tested on 3+ real people; insights in diary.md.
  • Handoff bundle ran through Claude Code and turned into live code in git.
  • The deck was read by at least one other person, and they said “got the point in 3 minutes.”
  • Wow element works in a mobile browser (on your phone).
